# 第十五讲AI提示词与测试清单

> 课程：`Java Web应用开发`  
> 对应课堂：`第十五讲 用户认证与权限控制`  
> 使用场景：`课堂实录`、`课外教学展示视频`、`创新成果报告支撑材料`

## 一、使用原则

本节课中，AI 的使用原则要始终明确：

1. AI 用于 `发现问题`，不是直接代替学生完成代码。
2. AI 用于 `知识点核对`，不是替代课程讲解。
3. AI 用于 `生成验证路径`，不是替代测试与复盘。
4. 所有 AI 输出都必须经过：
   - 学生判断
   - 教师引导
   - 系统验证

建议课堂开场直接说明：

## `AI 助教的任务不是帮你写答案，而是帮你更快地发现问题、对照知识点并生成验证方案。`

## 二、课堂可直接使用的 AI 提示词

下面的提示词按课堂阶段分类，可直接复制使用。

## 1. 教师总控提示词

适用于教师在投影端示范 AI 代码审查。

```text
请审查这个 Java Web 用户认证与权限控制模块，重点检查以下方面：

1. 认证流程是否正确
2. Session 的创建、保存、注销是否规范
3. 是否存在未授权访问或越权访问风险
4. RoleBasedAuthFilter 是否真正实现了基于角色的访问控制
5. JSP 菜单渲染是否与服务端权限控制一致
6. 密码处理是否存在安全隐患
7. 是否可能存在固定会话攻击、会话残留或重复提交问题

请按以下格式输出：
- 问题位置
- 风险说明
- 对应课程知识点
- 修改建议
- 建议验证方式
```

## 2. A组提示词：认证组

适用于审查 `LoginServlet`、`LogoutServlet`。

```text
请审查这个 Java Web 登录与登出模块，重点检查：

1. 登录成功后是否正确将用户信息保存到 Session
2. 登录失败时是否进行了合理反馈
3. 注销时是否真正清除了登录状态
4. 是否存在会话残留问题
5. 是否存在用户重新登录后状态混乱的问题

请按以下格式输出：
- 问题位置
- 风险说明
- 对应知识点（Authentication / Session-Based Login / Logout）
- 修改建议
- 建议验证方式
```

## 3. B组提示词：授权组

适用于审查 `RoleBasedAuthFilter`。

```text
请审查这个 Java Web 角色权限过滤器，重点检查：

1. 是否只判断了“已登录”，而没有判断角色
2. 是否正确识别 ADMIN 用户
3. 普通用户是否仍可能访问 /admin/* 路径
4. 未登录访问是否被正确拦截
5. 是否存在授权逻辑过宽或判断遗漏

请按以下格式输出：
- 问题位置
- 风险说明
- 对应知识点（Authorization / RBAC / Filter）
- 修改建议
- 建议验证方式
```

## 4. C组提示词：视图一致性组

适用于审查 JSP 菜单、按钮显示逻辑。

```text
请审查这个 Java Web JSP 页面中的菜单显示逻辑，重点检查：

1. 是否根据 sessionScope.user 正确显示登录状态
2. 是否根据角色正确显示“管理后台”入口
3. 前端菜单隐藏是否与服务端权限控制一致
4. 是否存在“前端隐藏了入口，但直接输入 URL 仍可访问”的风险

请按以下格式输出：
- 问题位置
- 风险说明
- 对应知识点（View Rendering / Session / Authorization）
- 修改建议
- 建议验证方式
```

## 5. 知识点对齐提示词

适用于把 AI 输出与课堂知识点对应起来。

```text
请根据以下 Java Web 课堂知识点，判断这段代码中哪些知识点已经正确体现，哪些存在问题：

知识点清单：
1. Authentication
2. Authorization
3. Session-Based Login
4. RBAC
5. Filter
6. Password Hash

请按以下格式输出：
- 知识点名称
- 当前情况（已正确体现 / 部分体现 / 存在问题）
- 代码依据
- 改进建议
```

## 6. 修改建议精炼提示词

适用于学生已经发现问题，但希望 AI 把建议说得更清楚。

```text
我已经知道这个 Java Web 权限控制模块存在问题。请不要直接重写整段代码，而是只告诉我：

1. 最优先应修改的 3 个点
2. 每个修改点的理由
3. 每个修改点对应的课程知识点
4. 修改后必须验证的系统行为
```

## 7. 教师纠偏提示词

适用于 AI 说得过多、过杂时，帮助教师快速收束。

```text
请把这个 Java Web 认证与授权问题的分析结果压缩为课堂上最值得强调的 3 个问题，每个问题只保留：

1. 问题本身
2. 为什么危险
3. 课堂上最应该提醒学生的知识点
```

## 三、课堂可直接使用的 AI 测试提示词

## 1. 最小验证清单生成提示词

```text
请为这个 Java Web 登录与权限控制模块生成一个最小验证清单，要求覆盖以下场景：

1. 未登录访问后台
2. 普通用户访问后台
3. 管理员访问后台
4. 注销后再次访问后台
5. JSP 菜单显示与角色一致性

请按“测试步骤 + 预期结果”格式输出，尽量简洁。
```

## 2. 手工测试用例生成提示词

```text
请为这个 Java Web 用户认证与权限控制模块生成 6 个手工测试用例，覆盖：

1. 成功登录
2. 登录失败
3. 未登录拦截
4. 普通用户越权访问
5. 管理员正常访问
6. 退出登录后访问受限资源

请按以下格式输出：
- 测试编号
- 测试目标
- 操作步骤
- 预期结果
```

## 3. 修改后回归测试提示词

```text
请根据这个 Java Web 权限控制模块的修改内容，生成一个回归测试清单，重点检查：

1. 修改后是否修复原问题
2. 是否引入新的访问异常
3. 菜单显示与后端控制是否仍一致
4. 登出是否真正失效

输出格式：
- 回归点
- 测试步骤
- 预期结果
```

## 四、课堂建议使用的手工测试清单

下面这份不用 AI 也能直接使用，建议打印出来做课堂任务单。

## 测试清单 A：核心权限验证

| 编号 | 测试目标 | 测试步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| T1 | 未登录拦截 | 直接访问 `/admin/panel` | 被重定向到登录页，或返回 403 |
| T2 | 普通用户访问后台 | 使用 `tom / USER` 登录后访问 `/admin/panel` | 被拒绝访问，不能进入后台 |
| T3 | 管理员访问后台 | 使用 `admin / ADMIN` 登录后访问 `/admin/panel` | 正常进入后台 |
| T4 | 登出后失效 | 登录后点击退出，再访问 `/admin/panel` | 再次被拦截 |
| T5 | 登录失败反馈 | 输入错误密码登录 | 显示“用户名或密码错误”等提示 |
| T6 | 菜单一致性 | 分别以未登录、USER、ADMIN 三种状态查看页面菜单 | 菜单显示与权限规则一致 |

## 测试清单 B：视图与后端一致性验证

| 编号 | 测试目标 | 测试步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| V1 | 未登录菜单 | 未登录状态打开首页 | 不显示“管理后台”入口 |
| V2 | 普通用户菜单 | `tom / USER` 登录后打开首页 | 不显示“管理后台”入口 |
| V3 | 管理员菜单 | `admin / ADMIN` 登录后打开首页 | 显示“管理后台”入口 |
| V4 | 隐藏入口绕过验证 | 普通用户在地址栏直接输入 `/admin/panel` | 仍被拒绝访问 |

## 测试清单 C：会话行为验证

| 编号 | 测试目标 | 测试步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| S1 | 登录会话建立 | 登录成功后刷新首页 | 页面仍显示当前用户信息 |
| S2 | 注销会话失效 | 登出后刷新页面 | 不再显示登录用户信息 |
| S3 | 会话角色生效 | 不同角色登录后访问相同资源 | 权限行为不同 |

## 七、课堂中教师可直接使用的评价话术

## 1. 针对 AI 使用方式

- “你们不是在抄 AI 的建议，而是在判断 AI 的建议是否符合课程知识和系统行为。”

## 2. 针对知识点区分

- “这不是简单的登录问题，而是认证之后的授权问题。用户已经登录，不代表就有后台权限。”

## 3. 针对视图层误区

- “菜单隐藏只是前端体验控制，不是安全控制。真正的权限边界必须落在服务端。”

## 4. 针对测试闭环

- “修复不是写完代码就结束，而是必须通过不同角色路径验证系统行为。”

## 九、如果 AI 输出不理想，如何追问

课堂上 AI 有时会输出过泛或过度展开，建议这样追问。

## 1. 当 AI 太空泛

```text
请不要泛泛而谈，只指出当前代码中最关键的 2 个安全问题，并分别说明：
1. 为什么危险
2. 该如何验证问题是否存在
```

## 2. 当 AI 直接给整段代码

```text
请不要直接重写整段代码，只指出需要修改的具体位置、修改理由和最小修改方案。
```

## 3. 当 AI 没有对应知识点

```text
请把你的分析结果与以下知识点逐项对应：
Authentication、Authorization、Session-Based Login、RBAC、Filter、Password Hash
```

## 4. 当 AI 忽略验证

```text
请为你提出的每个修改建议补充一个最小验证步骤和预期结果。
```
