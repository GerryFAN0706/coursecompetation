## "一半成功"的灾难

###### The "Half-Success" Disaster

**场景：下单操作 (Scenario: Place Order)**

```java
// 步骤1: 扣减库存 ✅成功
inventoryDao.decrease(productId, qty);

// 步骤2: 创建订单 ❌失败（数据库异常）
orderDao.insert(orderId, userId, productId);
```

**结果 (Result)**:
- ✅ 库存已减少
- ❌ 订单未创建
- 💥 数据不一致！

**问题**: 两个操作应该**要么都成功，要么都失败**  
*Problem: Both should succeed or both should fail*

## 事务的必要性

###### Need for Transactions

**什么是事务？** *What is a transaction?*

> 将多个数据库操作作为一个**不可分割的单元**  
> 要么全部成功，要么全部回滚  
> *Multiple operations as one **indivisible unit***

**ACID特性 (ACID Properties)**:

<!-- _class: cols2_ol_ci -->

- **A-Atomicity** 原子性  
  *All or nothing*
- **C-Consistency** 一致性  
  *Data integrity*
- **I-Isolation** 隔离性  
  *Concurrent safety*
- **D-Durability** 持久性  
  *Permanent storage*

## JDBC事务控制

###### JDBC Transaction Control

**核心API (Core APIs)**:

```java
Connection conn = dataSource.getConnection();

// 1. 关闭自动提交 (Disable auto-commit)
conn.setAutoCommit(false);

// 2. 执行多个操作
dao1.operation1(conn);
dao2.operation2(conn);

// 3. 提交事务 (Commit)
conn.commit();

// 4. 异常时回滚 (Rollback on error)
conn.rollback();
```

**关键**: 多个DAO共用**同一个connection**

## Service层事务模板

###### Service Layer Transaction Template

**标准事务处理模式** *Standard transaction pattern*

```java
public class OrderService {
    private OrderDao orderDao = new JdbcOrderDao();
    private InventoryDao inventoryDao = new JdbcInventoryDao();
    
    public void placeOrder(Long userId, Long productId, int qty) {
        Connection conn = null;
        try {
            conn = DBUtil.getConnection();
            conn.setAutoCommit(false);  // 开启事务
            
            inventoryDao.decrease(conn, productId, qty);  // 扣库存
            orderDao.insert(conn, userId, productId);     // 创建订单
            
            conn.commit();  // 提交
        } catch (Exception e) {
            if (conn != null) conn.rollback();  // 回滚
            throw new RuntimeException("下单失败", e);
        } finally {
            if (conn != null) conn.setAutoCommit(true);  // 恢复
            if (conn != null) conn.close();
        }
    }
}
```

## 事务隔离级别

###### Transaction Isolation Levels

**四种隔离级别 (Four Levels)**:

| 级别 | 说明 | 问题 |
|------|------|------|
| **READ_UNCOMMITTED** | 读未提交 *Read uncommitted* | 脏读 *Dirty read* |
| **READ_COMMITTED** | 读已提交 *Read committed* | 不可重复读 *Non-repeatable* |
| **REPEATABLE_READ** | 可重复读 *Repeatable* | 幻读 *Phantom read* |
| **SERIALIZABLE** | 串行化 *Serializable* | 性能低 *Low performance* |

**推荐**: `READ_COMMITTED`（MySQL默认：`REPEATABLE_READ`）

## 为什么需要连接池？

###### Why Connection Pool?

**传统方式问题 (Problems)**:

<!-- _class: cols2_ol_ci -->

- **每次创建** 开销大  
  *High overhead*
- **无法复用** 浪费资源  
  *No reuse*
- **高并发** 性能瓶颈  
  *Performance bottleneck*
- **连接限制** 数据库压力大  
  *DB pressure*

**连接池方案 (Pool Solution)**:

<!-- _class: cols2_ol_ci -->

- **预创建** 启动时创建多个  
  *Pre-create*
- **可复用** 用完归还  
  *Reusable*
- **高效** 避免重复创建  
  *Efficient*
- **可控** 限制最大连接数  
  *Controllable*

## Druid连接池

###### Druid Connection Pool

**阿里巴巴开源连接池** *Alibaba open-source pool*

**核心配置参数 (Core Parameters)**:

<!-- _class: cols2_ol_ci -->

- **initialSize** 初始连接数  
  *Initial size*
- **maxActive** 最大连接数  
  *Max connections*
- **minIdle** 最小空闲数  
  *Min idle*
- **maxWait** 最大等待时间  
  *Max wait time*

**druid.properties示例**:
```properties
driverClassName=com.mysql.cj.jdbc.Driver
url=jdbc:mysql://localhost:3306/demo
username=root
password=123456
initialSize=5
maxActive=20
```

## 使用Druid

###### Using Druid

**初始化连接池 (Initialize Pool)**:

```java
public class DataSourceUtil {
    private static DataSource dataSource;
    
    static {
        Properties props = new Properties();
        props.load(/* 加载druid.properties */);
        dataSource = DruidDataSourceFactory.createDataSource(props);
    }
    
    public static Connection getConnection() throws SQLException {
        return dataSource.getConnection();
    }
}
```

**使用 (Usage)**:
```java
// 获取连接（来自连接池）
Connection conn = DataSourceUtil.getConnection();
// 使用完毕，归还到池中
conn.close();  // 不是真正关闭，而是归还！
```

## 批处理优化

###### Batch Processing

**批量操作提升性能 (Batch for Performance)**:

```java
public void batchInsert(List<User> users) {
    String sql = "INSERT INTO users (name, age) VALUES (?, ?)";
    Connection conn = DBUtil.getConnection();
    PreparedStatement ps = conn.prepareStatement(sql);
    
    for (User user : users) {
        ps.setString(1, user.getName());
        ps.setInt(2, user.getAge());
        ps.addBatch();  // 添加到批次
    }
    
    ps.executeBatch();  // 批量执行
}
```

**性能对比**: 批处理比逐条执行快10-100倍

## 事务保存点

###### Transaction Savepoint

**部分回滚功能 (Partial Rollback)**:

```java
Connection conn = DBUtil.getConnection();
conn.setAutoCommit(false);

Savepoint sp1 = conn.setSavepoint();  // 设置保存点

try {
    // 执行操作1
    dao.operation1(conn);
    
    Savepoint sp2 = conn.setSavepoint();  // 第二个保存点
    
    // 执行操作2
    dao.operation2(conn);  // 如果失败
    
} catch (Exception e) {
    conn.rollback(sp1);  // 回滚到保存点sp1
}
```

## 课程思政：诚信与契约

###### Ideological Education: Integrity & Contract

### 从事务到社会契约 (From Transaction to Social Contract)

**事务的本质 (Essence of Transaction)**:
- ✅ 要么全部完成 *All or nothing*
- ✅ 保证一致性 *Consistency*
- ✅ 不能部分成功 *No partial success*

**社会契约类比 (Social Contract Analogy)**:

**商业诚信 (Business Integrity)**:
> 交易要么达成，要么撤销  
> 不能"收了钱不发货"或"发了货不给钱"  
> *Transaction completes or cancels, no half-way*

**社会秩序 (Social Order)**:
- 契约精神维护公平 *Contract maintains fairness*
- 规则保障各方权益 *Rules protect rights*
- 诚信是社会基石 *Integrity is foundation*
