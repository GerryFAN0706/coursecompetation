## Controller：控制器详解

###### Controller Explained

**Servlet作为Controller** *Servlet as Controller*

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, ...) {
        // 1. 接收参数 (Receive parameters)
        String username = request.getParameter("username");
        String password = request.getParameter("password");
        
        // 2. 调用Model处理业务 (Call Model)
        UserService service = new UserService();
        User user = service.login(username, password);
        
        // 3. 准备数据，转发到View (Forward to View)
        if (user != null) {
            request.setAttribute("user", user);
            request.getRequestDispatcher("/welcome.jsp").forward(...);
        } else {
            request.setAttribute("error", "登录失败");
            request.getRequestDispatcher("/login.jsp").forward(...);
        }
    }
}
```

## View：视图层详解

###### View Explained

**JSP作为View** *JSP as View*

```jsp
<%@ page contentType="text/html;charset=UTF-8" %>
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<html>
<body>
    <h1>欢迎 (Welcome)</h1>
    
    <c:if test="${user != null}">
        <p>用户名：${user.username}</p>
        <p>ID：${user.id}</p>
    </c:if>
    
    <c:if test="${error != null}">
        <p style="color:red;">${error}</p>
    </c:if>
</body>
</html>
```

**特点**: 只负责显示，不处理业务  
*Feature: Display only, no business logic*

## MVC完整数据流

###### Complete MVC Data Flow

**从请求到响应 (From Request to Response)**:

```
1. Browser → HTTP请求 → Controller (Servlet)
   ↓
2. Controller → 调用 → Model (Service/DAO)
   ↓
3. Model → 处理业务 → 返回数据
   ↓
4. Controller → setAttribute → 转发 → View (JSP)
   ↓
5. View → 取数据 (EL) → 渲染页面
   ↓
6. View → HTML响应 → Browser
```

## 请求转发 Forward

###### Request Forward

**语法 (Syntax)**:
```java
request.getRequestDispatcher("/page.jsp").forward(request, response);
```

**特点 (Features)**:

<!-- _class: cols2_ol_ci -->

- **内部跳转** 服务器内部  
  *Internal jump*
- **一次请求** 只有一次  
  *One request*
- **URL不变** 地址栏不变  
  *URL unchanged*
- **共享数据** request共享  
  *Share request*

**使用场景**: Controller → View传递数据

## 重定向 Redirect

###### Redirect

**语法 (Syntax)**:
```java
response.sendRedirect("page.jsp");
```

**特点 (Features)**:

<!-- _class: cols2_ol_ci -->

- **外部跳转** 浏览器发起  
  *External jump*
- **两次请求** 两个请求  
  *Two requests*
- **URL改变** 地址栏改变  
  *URL changes*
- **数据不共享** 新request  
  *New request*

**使用场景**: 防止表单重复提交

## Forward vs Redirect对比

###### Forward vs Redirect Comparison

**场景选择 (Scenario Selection)**:

<!-- _class: cols-2 -->

<div class="ldiv">

**使用Forward** ✅

**场景 (Scenarios)**:
- Servlet → JSP
- 需要传递数据
- 查询操作后显示

**示例**:
```java
// 查询后显示
User u = service.find(id);
request.setAttribute("user", u);
request.getRequestDispatcher(
  "/view.jsp").forward(...);
```

</div>

<div class="rdiv">

**使用Redirect** ✅

**场景 (Scenarios)**:
- 操作完成后跳转
- 防止重复提交
- 跨应用跳转

**示例**:
```java
// 添加后重定向
service.addUser(user);
response.sendRedirect(
  "success.jsp");
```

</div>

## MVC架构演进

###### MVC Architecture Evolution

**JSP Model 1 vs Model 2**

<!-- _class: cols-2 -->

<div class="ldiv">

**Model 1** (旧模式)

```
Browser → JSP
  ├─ 接收请求
  ├─ 处理业务
  ├─ 访问数据库
  └─ 生成页面
```

**问题**:
- ❌ JSP职责过重
- ❌ 代码混乱
- ❌ 难以维护

</div>

<div class="rdiv">

**Model 2** (MVC)

```
Browser → Servlet → Model
            ↓
          JSP显示
```

**优势**:
- ✅ 职责分离
- ✅ 结构清晰
- ✅ 易于维护

</div>

## Forward vs Redirect总结

###### Forward vs Redirect Summary

**对比表格 (Comparison Table)**:

| 特性 | Forward | Redirect |
|------|---------|----------|
| **请求次数** | 1次 *One* | 2次 *Two* |
| **URL变化** | 不变 *Unchanged* | 改变 *Changed* |
