## 一、实验目标

1. 防御固定会话攻击（Session Fixation）
2. 实现Session超时配置
3. 实现"记住我"（Remember Me）功能

## 二、安全威胁分析

### 固定会话攻击原理
```
攻击者 → 获取一个合法的JSESSIONID（如访问登录页）
攻击者 → 将该JSESSIONID植入受害者浏览器（如通过URL）
受害者 → 使用该JSESSIONID登录
攻击者 → 使用同一JSESSIONID访问系统 → 以受害者身份操作
```

**防御**：登录成功后，销毁旧Session，创建新Session，迁移必要属性。

## 三、核心实现

### 3.1 防固定会话攻击（LoginServlet改造）

```java
@Override
protected void doPost(HttpServletRequest req, HttpServletResponse resp)
        throws ServletException, IOException {
    req.setCharacterEncoding("UTF-8");
    String username = req.getParameter("username");
    String password = req.getParameter("password");

    User user = userService.login(username, password);
    if (user != null) {
        // 防固定会话攻击：销毁旧Session，创建新Session
        HttpSession oldSession = req.getSession(false);
        if (oldSession != null) {
            oldSession.invalidate();
        }
        HttpSession newSession = req.getSession(true);
        newSession.setAttribute("user", user);
        newSession.setMaxInactiveInterval(30 * 60); // 30分钟超时

        // 处理"记住我"
        String rememberMe = req.getParameter("rememberMe");
        if ("on".equals(rememberMe)) {
            String token = generateRememberToken(user.getUsername());
            saveRememberToken(user.getUsername(), token);
            Cookie cookie = new Cookie("remember_token", token);
            cookie.setMaxAge(7 * 24 * 60 * 60); // 7天
            cookie.setHttpOnly(true);
            cookie.setPath(req.getContextPath());
            resp.addCookie(cookie);
        }

        resp.sendRedirect(req.getContextPath() + "/index.jsp");
        return;
    }

    req.setAttribute("error", "用户名或密码错误");
    req.getRequestDispatcher("/login.jsp").forward(req, resp);
}
```

### 3.2 Remember Token生成

```java
private String generateRememberToken(String username) {
    String raw = username + System.currentTimeMillis() + UUID.randomUUID();
    return PasswordUtil.sha256(raw);
}

// 简化实现：使用内存Map存储（生产环境应存数据库）
private static final Map<String, String> REMEMBER_TOKENS =
    new ConcurrentHashMap<>();

private void saveRememberToken(String username, String token) {
    REMEMBER_TOKENS.put(token, username);
}

public static String getUsernameByToken(String token) {
    return REMEMBER_TOKENS.get(token);
}
```

### 3.4 自动登录过滤器（RememberMeFilter）

```java
@WebFilter("/*")
public class RememberMeFilter implements Filter {
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpSession session = request.getSession(false);

        // 如果已登录，直接放行
        if (session != null && session.getAttribute("user") != null) {
            chain.doFilter(req, res);
            return;
        }

        // 检查remember_token Cookie
        Cookie[] cookies = request.getCookies();
        if (cookies != null) {
            for (Cookie cookie : cookies) {
                if ("remember_token".equals(cookie.getName())) {
                    String username = LoginServlet.getUsernameByToken(cookie.getValue());
                    if (username != null) {
                        User user = new UserDao().findByUsername(username);
                        if (user != null) {
                            HttpSession newSession = request.getSession(true);
                            newSession.setAttribute("user", user);
                        }
                    }
                    break;
                }
            }
        }
        chain.doFilter(req, res);
    }
}
```

## 四、测试结果

| 测试场景 | 操作 | 预期 | 实际 | 通过 |
|:---|:---|:---|:---|:---|
| 固定会话防御 | 登录前后对比JSESSIONID | ID不同 | 登录后JSESSIONID已更换 | ✓ |
| Session超时 | 登录后等待31分钟 | Session过期需重新登录 | 31分钟后访问提示登录 | ✓ |
| 记住我-勾选 | 勾选"记住我"登录 | 关闭浏览器后重新打开仍登录 | Cookie有效，自动登录 | ✓ |
| 记住我-未勾选 | 不勾选"记住我"登录 | 关闭浏览器后需重新登录 | 无remember Cookie | ✓ |
| 记住我-退出 | 登录后点退出 | remember Cookie被清除 | Cookie已清除 | ✓ |
| HttpOnly | 浏览器控制台尝试读取Cookie | document.cookie无法读取 | HttpOnly生效 | ✓ |

## 六、实验总结

Session安全看似简单，实际涉及多个攻击面。最重要的防御原则是：
1. **登录后更换Session ID** — 防固定会话攻击
2. **合理设置超时** — 防遗忘退出导致的会话劫持
3. **Remember Me Token必须随机且不可预测** — 防Token伪造
4. **Cookie设置HttpOnly** — 防XSS窃取Cookie

这些都是"看不见的安全"——用户感知不到，但缺少任何一项都可能导致严重的安全事故。
