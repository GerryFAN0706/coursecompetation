# Workshop 网站内容映射到人工智能赛道材料

> 适用对象：已建设 `AI辅助教学案例` 网站，拟申报 `第六届全国高校教师教学创新大赛人工智能赛道`  
> 核心目标：把公开网站中的案例内容，转化为可用于 `创新成果报告`、`视频材料`、`证明材料`、`现场汇报` 的比赛材料

## 一、总体判断

你的网站内容对人工智能赛道是 `有价值且可用` 的，但要注意：

1. 网站本身更像 `公开展示与经验分享平台`。
2. 比赛评审更关注 `真实课程中的教学闭环`，而不是单纯展示“我会用哪些 AI 工具”。
3. 因此，网站内容最适合做两件事：
   - 提炼为参赛材料中的 `创新主线`
   - 作为 `辅助支撑证据`

最适合你作为人工智能赛道申报主线的，不是单个工具，而是下面这条闭环：

`课前学情分析与分层 -> AI生成个性化任务/资源 -> 课中人机协同学习 -> 课后智能评价与反馈 -> 学生迭代改进 -> 教师基于数据再次干预`

## 二、最适合采用的参赛主线

结合你的网站内容，建议将参赛课程的人工智能创新模式概括为：

## `人工智能赋能 Java Web 课程项目式教学的闭环教学模式`

可分为三段：

### 1. 课前：个性化任务生成与学情诊断

对应网站内容：

- `个性化实验设计生成器`
- 学生分层设计
- AI 生成实验手册、评分标准、测试用例

可提炼的比赛表达：

1. 基于历史作业、测验成绩、知识点掌握情况，对学生进行分层。
2. 借助生成式人工智能，为不同层次学生生成 `同知识点、异场景、分层次` 的实验任务。
3. 在实验开始前即完成“难度适配 + 防抄袭 + 目标对齐”。

### 2. 课中：智能代码审查与人机协同教学

对应网站内容：

- `智能代码审查助手`
- Cursor 辅助代码理解、错误诊断、知识点检查
- 测试用例驱动的功能验证

可提炼的比赛表达：

1. 学生在真实项目开发过程中使用 AI 工具进行代码理解、错误定位和代码优化。
2. 教师不再只做结果评分，而是借助 AI + 测试用例 + 知识点对照，对学生进行过程性支持。
3. 形成“教师主导、学生主体、AI协同”的课堂教学模式。

### 3. 课后：智能反馈与持续改进

对应网站内容：

- AI 生成反馈报告
- 学生二次迭代提交
- 教师根据结果进行后续干预

可提炼的比赛表达：

1. 将 AI 反馈结果转化为可执行的修改建议。
2. 学生基于反馈进行二次修正，实现形成性学习。
3. 教师依据数据对共性薄弱点进行再教学和精准指导。

## 六、哪些网站内容需要补证据后才能用于比赛

虽然网站内容已经很强，但以下信息若要用于正式参赛，建议补充更严谨的课程证据。

## 1. 数据指标需要补口径说明

例如：

- 调试时间减少 40%
- 教师审查时间减少 67%
- 代码质量提升 30%
- 规范性提升 37%

比赛中最好说明：

1. 样本量是多少
2. 统计周期是什么
3. 对比对象是谁
4. 评价标准如何设定

## 2. 需要绑定到具体课程和具体班级

建议补充：

1. 课程名称
2. 开课学期
3. 授课对象
4. 班级人数
5. AI 工具使用频率
6. 使用的具体教学环节

## 3. 需要体现“数据驱动”的证据

建议补充：

1. 学生分层依据
2. 学情诊断表
3. 后台日志
4. 测试结果统计
5. 学生使用记录
6. 前后测对比

## 4. 需要体现学术诚信与伦理安排

人工智能赛道非常看重这一点，建议补充：

1. 学生使用 AI 的边界规定
2. 禁止事项
3. 教师人工复核机制
