## 回顾Servlet的痛点

###### Recall Servlet's Pain Point

**Servlet中的"意大利面条"代码** *Spaghetti code in Servlet*

```java
out.println("<html><head><title>欢迎</title>");
out.println("<style>body { background: #f0f0f0; }</style>");
out.println("<h1>欢迎你, " + name + " 特工!</h1>");
out.println("<table><tr><td>" + name + "</td></tr>");
out.println("</table></body></html>");
```

**问题 (Problems)**:
- 😵 HTML和Java混在一起 *Mixed together*
- 😫 难以阅读修改 *Hard to read*
- 🤯 设计师无法编辑 *Designer can't edit*

## 设计师的困境

###### Designer's Dilemma

**角色 (Roles)**:
- 👩‍🎨 **小美** - 设计HTML *Designer Mei*
- 👨‍💻 **小明** - 写Servlet *Developer Ming*

**问题流程 (Problem Flow)**:

<!-- _class: cols2_ol_ci -->

- **Step 1** 小美设计HTML  
  *Mei designs HTML*
- **Step 2** 小明复制到Servlet  
  *Ming copies to Servlet*
- **Step 3** 小美要改颜色  
  *Mei wants to change*
- **Step 4** 小明在Java里找HTML  
  *Ming searches in Java*
- **Step 5** 循环往复  
  *Repeat forever*
- **结果** 合作灾难！  
  *Disaster!*

## 问题根源

###### Root Cause

**Servlet的特点 (Servlet Characteristics)**:

<!-- _class: cols2_ol_ci -->

- **✅ 擅长** 业务逻辑  
  *Good at logic*
- **✅ 擅长** 流程控制  
  *Good at control*
- **❌ 不擅长** 生成HTML  
  *Bad at HTML*
- **❌ 不擅长** 页面设计  
  *Bad at design*

**思路转变 (New Idea)**:
> 在HTML里"挖坑"，用Java"填数据"  
> *Dig holes in HTML, fill with Java data*

## JSP：优雅的解决方案

###### JSP: The Elegant Solution

<!-- _class: bq-blue -->

> **什么是JSP？ (What is JSP?)**
>
> JSP (JavaServer Pages) 是一种以HTML为中心，允许嵌入Java代码的动态网页技术  
> *JSP is an HTML-centric technology that allows embedding Java code to create dynamic web pages*

**JSP的特点 (JSP Features)**:
- 🎨 **以HTML为主** (HTML-centric) - ☕ **Java为辅** (Java as supplement)
- 🔧 **易于维护** (Easy to maintain) - 👥 **前后端友好** (Frontend/Backend friendly)

**核心理念 (Core Philosophy)**:
> "前端的归前端，后端的归后端"   > *"Frontend to frontend, backend to backend"*

## JSP核心语法预览



**三种主要语法 (Three Main Syntaxes)**:

**1️⃣ JSP表达式 (Expression)**
```jsp
<%= 变量或表达式 %>  ← 用于输出 (For output)
```

**2️⃣ JSP脚本片段 (Scriptlet)**
```jsp
<% Java代码 %>  ← 用于逻辑 (For logic)
```

**3️⃣ JSP指令 (Directive)**
```jsp
<%@ 指令名 属性 %>  ← 用于配置 (For configuration)
```

## 使用JSP表达式

###### Using JSP Expressions

**示例代码 (Example)**:

```jsp
<%-- 获取数据 (Get data) --%>
<% String name = (String) request.getAttribute("agent"); %>

<%-- 显示数据 (Display data) --%>
<h3>欢迎，<%= name %></h3>
<p>时间：<%= new java.util.Date() %></p>
```

**语法说明 (Syntax)**:
- `<%= 表达式 %>` 输出结果
- 等价于 `out.print(表达式)`

## 完整JSP示例

###### Complete JSP Example

```jsp
<%@ page contentType="text/html;charset=UTF-8" %>
<html>
<head><title>任务简报</title></head>
<body>
    <% String name = (String) request.getAttribute("agent"); %>
    
    <h1>🎖️ 任务简报 (Mission Briefing)</h1>
    <h3>欢迎，<%= name %> 特工！</h3>
    <p>任务代号：<%= (int)(Math.random() * 1000) %></p>
    <p>时间：<%= new java.util.Date() %></p>
</body>
</html>
```

## JSP运行流程

###### JSP Execution Flow

**从访问到响应** *From access to response*

```
1. 用户访问 mission.jsp
   ↓
2. Tomcat检查是否第一次访问
   ↓
3. 翻译 mission.jsp → mission_jsp.java (Servlet)
   ↓
4. 编译 mission_jsp.java → mission_jsp.class
   ↓
5. 执行 mission_jsp.class
   ↓
6. 生成HTML响应 → 返回浏览器
```

**后续访问**: 直接执行已编译的 `.class` 文件 ⚡

## JSP表达式详解

###### JSP Expression Explained

**语法 (Syntax)**: `<%= 表达式 %>`

**使用场景 (Use Cases)**:

<!-- _class: cols2_ol_ci -->

- **输出变量** `<%= userName %>`  
  *Output variable*
- **对象属性** `<%= product.getName() %>`  
  *Object property*
- **计算** `<%= 5 * 10 %>`  
  *Calculate*
- **调用方法** `<%= request.getParameter("id") %>`  
  *Call method*
- **显示时间** `<%= new java.util.Date() %>`  
  *Display time*
- **三元运算** `<%= age > 18 ? "成年" : "未成年" %>`  
  *Ternary*

## JSP生命周期

**JSP的神奇转换** *JSP's Magic Transformation*
```
mission.jsp (JSP文件)
    ↓ 翻译 (Translation)
mission_jsp.java (Servlet源码)
    ↓ 编译 (Compilation)
mission_jsp.class (字节码)
    ↓ 执行 (Execution)
HTML响应 (HTML Response)
```

**关键理解 (Key Understanding)**:

<!-- _class: bq-blue -->

> **JSP本质上就是一种编写Servlet的便捷方式！**  *JSP is essentially a convenient way to write Servlets!*
>
> JSP让我们专注于页面**长什么样**，而不是**怎么生成**  *JSP lets us focus on **what** the page looks like, not **how** to generate it*

## Servlet改造示例

###### Servlet Modification Example

**LoginServlet.java** - 不再输出HTML

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request,
                         HttpServletResponse response) {
        // 1. 获取参数 (Get parameter)
        String name = request.getParameter("agentName");
        
        // 2. 准备数据 (Prepare data)
        request.setAttribute("agent", name);
        request.setAttribute("task", "精通JSP");
        
        // 3. 转发到JSP (Forward to JSP)
        request.getRequestDispatcher("/mission.jsp")
            .forward(request, response);
    }
}
```

## JSP vs Servlet 对比

###### JSP vs Servlet

<!-- _class: cols2_ol_ci -->

**对比要点 (Comparison)**:

- **Servlet擅长** 业务逻辑  
  *Logic control*
- **JSP擅长** 页面展示  
  *Page presentation*
- **Servlet** 纯Java代码  
  *Pure Java*
- **JSP** HTML为主  
  *HTML-centric*
- **Servlet** 维护难  
  *Hard to maintain*
- **JSP** 易于修改  
  *Easy to modify*

## 最佳实践：职责分离

###### Best Practice: Separation of Concerns

<!-- _class: bq-green -->

> **MVC模式预告 (MVC Pattern Preview)**
>
> - **Servlet** = Controller (控制器)  处理请求，控制流程
>
> - **JSP** = View (视图)     显示数据，呈现界面
>
> - **JavaBean** = Model (模型)     封装数据，业务对象

**核心原则 (Core Principle)**:
> 每个组件做自己最擅长的事  
> *Each component does what it does best*
