## 问题引入：一次网页请求的背后

###### Behind a Web Request

### 一个熟悉的场景 (A Familiar Scenario)

当我们在浏览器地址栏输入 `http://www.example.edu.cn` 并敲下回车...  
*When we type `http://www.example.edu.cn` in the browser and press Enter...*

**短短几秒内发生了什么？**  
*What happens in just a few seconds?*

1. 浏览器向服务器**发送请求** (Browser sends a **request**)
2. 服务器处理请求并**生成响应** (Server processes and **generates response**)
3. 浏览器接收响应并**渲染页面** (Browser receives and **renders the page**)

## HTTP协议 - Web的通用语

###### HTTP: Universal Language of the Web

### HTTP 请求三要素 (HTTP Request - Three Elements)

**客户端发送给服务器的"信件"**  
**1️⃣ 请求行 (Request Line)**
```http
GET /index.html HTTP/1.1
```
- `GET`: 请求方法 (Request Method)  `/index.html`: 请求路径 (Request Path/Resource)  `HTTP/1.1`: 协议版本 (Protocol Version)

**2️⃣ 请求头 (Request Headers)**
```http
Host: www.example.edu.cn
User-Agent: Chrome/120.0.0
Accept: text/html
```
- 附加信息，如主机名、浏览器类型等  *(Additional info like hostname, browser type, etc.)*
### HTTP 请求三要素 (HTTP Request - Three Elements)
**3️⃣ 请求体 (Request Body)**
- GET请求通常为空 (Usually empty for GET)
- POST请求包含提交数据 (POST contains submitted data)

## HTTP 请求与响应对比
<!-- _class: cols-2 -->

<div class="ldiv">

#### HTTP 请求 (Request)

**三个核心部分**:
1. **请求行 (Request Line)**
   - 方法Method + 路径Path + 协议版本
2. **请求头 (Headers)**
   - Host, User-Agent等
3. **请求体 (Body)**
   - POST数据

</div>

<div class="rdiv">

#### HTTP 响应 (Response)

**三个核心部分**:
1. **状态行 (Status Line)**
   - 协议版本 + 状态码Status Code
2. **响应头 (Headers)**
   - Content-Type等
3. **响应体 (Body)**
   - 实际HTML内容

</div>

## HTTP状态码

###### HTTP Status Codes

<!-- _class: cols2_ol_ci -->

**状态码分类 (Status Code Categories)**:

- **2xx 成功 (Success):**  
  `200 OK` - 请求成功
- **3xx 重定向 (Redirection):**  
  `301` 永久重定向, `302` 临时重定向
- **4xx 客户端错误 (Client Error):**  
  `404 Not Found`, `403 Forbidden`
- **5xx 服务器错误 (Server Error):**  
  `500 Internal Server Error`

## Tomcat - Web应用的家

###### Tomcat: Home for Web Applications

### 为什么需要Tomcat？ (Why Do We Need Tomcat?)

**问题 (Problem)**:
- **HTML** 文件: 浏览器可以直接打开  
  (Browsers can open HTML files directly)
- **包含Java代码的动态页面**: 浏览器**不能**直接执行Java代码  
  (Dynamic pages with Java: Browsers **cannot** execute Java code)

**结论 (Conclusion)**:
> 我们需要一个特殊的软件，它既能接收HTTP请求，又能运行Java代码。  
> (We need special software that can both receive HTTP requests and run Java code.)

这个"超级服务器"就是 **Servlet容器 (Servlet Container)**，Tomcat是其中最著名的一员。

## 什么是Tomcat？

###### What is Tomcat?

<!-- _class: bq-blue -->

> **Apache Tomcat**
>
> Apache基金会下的**开源、免费**的轻量级Web服务器和Servlet容器。  
> (An **open-source, free** lightweight web server and Servlet container from Apache Foundation.)
>
> **核心使命 (Core Mission)**:
> - 接收HTTP请求 (Receive HTTP requests)
> - 运行Java程序/Servlet (Run Java programs/Servlets)
> - 生成HTTP响应 (Generate HTTP responses)

## Tomcat 核心目录结构

###### Tomcat's Core Directory Structure

<!-- _class: cols2_ol_sq -->

**各司其职 (Each Has Its Own Duty)**:

- **`bin`** 📂 **启动器** (Launcher)  
  `startup.bat/.sh` - 启动服务器  
  `shutdown.bat/.sh` - 关闭服务器
- **`conf`** ⚙️ **配置中心** (Config Center)  
  `server.xml` - 服务器配置  
  `web.xml` - Web应用配置
- **`webapps`** 🏢 **应用目录** (Apps Directory)  
  存放所有Web应用（每个应用一个文件夹）  
  *All web applications go here*
- **`logs`** 📋 **日志目录** (Logs Directory)  
  `catalina.out` - 运行日志  
  排错必备 (Essential for debugging)

## 启动Tomcat服务器

**启动步骤 (Startup Steps)**:

**Windows系统**:
1. 进入 `bin` 目录 (Go to `bin` directory)
2. 双击运行 `startup.bat`
3. 看到控制台窗口 (Console window appears)
4. 等待启动完成 (Wait for startup to complete)
5. 浏览器访问: `http://localhost:8080`

**成功标志 (Success Indicator)**:
- ✅ 看到Tomcat欢迎页面  ✅ 控制台无报错信息 ✅ 端口8080正常监听

**常见问题 (Common Issues)**:
⚠️ 端口8080被占用 → 修改 `conf/server.xml` ⚠️ Java未安装 → 安装JDK并配置JAVA_HOME

## 目录结构示例

###### Directory Structure Example

**标准Web应用结构** *Standard web application structure*

```
webapps/
  └── myapp/              ← 应用根目录 (App root)
      ├── index.html      ← 主页 (Homepage)
      ├── WEB-INF/        ← 受保护目录 (Protected)
      │   ├── web.xml     ← 配置文件 (Config)
      │   ├── classes/    ← Java类 (Classes)
      │   └── lib/        ← 库文件 (Libraries)
      └── css/            ← 样式文件 (Styles)
```

**访问规则 (Access Rules)**:
- ✅ `index.html` 可直接访问
- ❌ `WEB-INF/` 目录无法直接访问（受保护）
