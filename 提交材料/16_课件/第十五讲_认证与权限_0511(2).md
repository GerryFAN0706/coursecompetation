## 认证与授权

###### Authentication & Authorization

**两个不同的概念 (Two Different Concepts)**:

<!-- _class: cols-2 -->

<div class="ldiv">

**认证 (Authentication)**

**问题**: 你是谁？
*Who are you?*

**过程**:
- 用户提供凭证（用户名+密码）
- 系统验证身份
- 确认用户身份

**比喻**: 身份证检查
*ID verification*

</div>

<div class="rdiv">

**授权 (Authorization)**

**问题**: 你能做什么？
*What can you do?*

**过程**:
- 检查用户角色/权限
- 判断是否有访问权限
- 允许或拒绝访问

**比喻**: 门禁卡权限
*Access card permission*

</div>

**顺序**: 先认证，后授权 | **核心**: 登录成功 ≠ 拥有所有权限

## Session认证完整流程

###### Session-Based Authentication Flow

**四个组件如何协同工作 (How Four Components Work Together)**:

<!-- _class: cols-2 -->

<div class="ldiv">

**请求流程 (Request Flow)**:

1. 用户提交登录表单
   *User submits login form*
2. **LoginServlet** 验证用户名密码
   *Verify credentials*
3. 验证通过 → 创建Session，存储User对象
   *Create session, store user object*
4. 后续每次请求 → **Filter** 拦截检查角色
   *Filter intercepts, checks role*
5. 角色匹配 → 放行；不匹配 → 403
   *Match → allow; Mismatch → deny*

</div>

<div class="rdiv">

**组件职责 (Component Responsibilities)**:

| 组件 | 职责 | 阶段 |
|:---|:---|:---|
| **LoginServlet** | 验证身份 | 认证 |
| **Session** | 维持登录状态 | 状态管理 |
| **Filter** | 拦截检查权限 | 授权 |
| **JSP** | 按角色渲染界面 | 展示 |

**核心原则**: 每个组件只做一件事
*Single Responsibility Principle*

</div>

## LoginServlet · 登录实现

###### LoginServlet · Authentication Implementation

**登录核心逻辑 (Login Core Logic)**:

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    protected void doPost(HttpServletRequest req, HttpServletResponse resp)
            throws IOException, ServletException {
        String username = req.getParameter("username");
        String password = req.getParameter("password");
        User user = userService.authenticate(username, password);
        if (user != null) {
            HttpSession session = req.getSession();
            session.setAttribute("user", user);   // 只存一个对象
            resp.sendRedirect(req.getContextPath() + "/index.jsp");
        } else {
            req.setAttribute("error", "用户名或密码错误");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
        }
    }
}
```

**要点 (Key Points)**:
- 只存一个`user`对象到Session，角色信息随User一起存入
- 登录成功用**重定向**，失败用**转发**（保留错误信息）

## 登出与Session安全

**安全登出 (Secure Logout)**:

```java
@WebServlet("/logout")
public class LogoutServlet extends HttpServlet {
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws IOException {
        HttpSession session = req.getSession(false);
        if (session != null) {
            session.invalidate();  // 彻底销毁Session
        }
        resp.sendRedirect(req.getContextPath() + "/login.jsp");
        }}
```

**关键区别 (Key Difference)**:
- `removeAttribute("user")` — 只删一个属性，Session仍存活
- `invalidate()` — **彻底销毁**整个Session，所有属性一并清除

## JSP按角色显示菜单

###### Role-Based Menu Display

**动态菜单 (Dynamic Menu)**:

```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<c:if test="${sessionScope.user != null}">
    <p>欢迎，${sessionScope.user.displayName}</p>
    <a href="${pageContext.request.contextPath}/logout">退出</a>
</c:if>
<c:if test="${sessionScope.user.role == 'ADMIN'}">
    <a href="${pageContext.request.contextPath}/admin/panel.jsp">管理后台</a>
</c:if>
```

**核心原则 (Core Principle)**:

> 前端隐藏菜单 = 用户体验优化（UX） 后端Filter拦截 = **真正的安全边界**
> **完整方案 = 前端隐藏 + 后端控制**

## 权限过滤器修复后代码

###### Authorization Filter (Fixed)

**基于角色的Filter** *Role-based filter*

```java
@WebFilter("/admin/*")
public class RoleBasedAuthFilter implements Filter {
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        HttpSession session = request.getSession(false);
        if (session == null) {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
            return;
        }
        User user = (User) session.getAttribute("user");
        if (user != null && "ADMIN".equals(user.getRole())) {
            chain.doFilter(req, res);   // ADMIN放行
        } else if (user != null) {
            response.sendError(403);    // 非ADMIN → 403
        } else {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }
}
```
