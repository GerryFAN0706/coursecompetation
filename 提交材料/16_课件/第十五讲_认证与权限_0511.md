## Filter权限检查 · 核心讲解

###### Role-Based Auth Filter · Key Explanation

**修复方案——基于角色的Filter** *Role-based filter*:

```java
@WebFilter("/admin/*")
public class RoleBasedAuthFilter implements Filter {
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        HttpSession session = request.getSession(false);

        if (session == null) {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
            return;
        }
        User user = (User) session.getAttribute("user");
        if (user != null && "ADMIN".equals(user.getRole())) {
            chain.doFilter(req, res);   // ADMIN放行
        } else if (user != null) {
            response.sendError(403);    // 非ADMIN → 403
        } else {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }
}
```

**关键**: 不是检查"是否登录"，而是检查"角色是否匹配"！

## 认证与授权

###### Authentication & Authorization

**两个不同的概念 (Two Different Concepts)**:

<!-- _class: cols-2 -->

<div class="ldiv">

**认证 (Authentication)**

**问题**: 你是谁？
*Who are you?*

**过程**:
- 用户提供凭证（用户名+密码）
- 系统验证身份
- 确认用户身份

**比喻**: 身份证检查
*ID verification*

</div>

<div class="rdiv">

**授权 (Authorization)**

**问题**: 你能做什么？
*What can you do?*

**过程**:
- 检查用户角色/权限
- 判断是否有访问权限
- 允许或拒绝访问

**比喻**: 门禁卡权限
*Access card permission*

</div>

**顺序**: 先认证，后授权 | **核心**: 登录成功 ≠ 拥有所有权限

## 登出与Session安全

###### Logout & Session Security

**安全登出 (Secure Logout)**:

```java
@WebServlet("/logout")
public class LogoutServlet extends HttpServlet {
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws IOException {
        HttpSession session = req.getSession(false);
        if (session != null) {
            session.invalidate();  // 彻底销毁Session
        }
        resp.sendRedirect(req.getContextPath() + "/login.jsp");
    }
}
```

**关键区别 (Key Difference)**:
- `removeAttribute("user")` — 只删一个属性，Session仍存活
- `invalidate()` — **彻底销毁**整个Session，所有属性一并清除

## JSP按角色显示菜单

###### Role-Based Menu Display

**动态菜单 (Dynamic Menu)**:

```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

<c:if test="${sessionScope.user != null}">
    <p>欢迎，${sessionScope.user.displayName}</p>
    <a href="${pageContext.request.contextPath}/logout">退出</a>
</c:if>

<c:if test="${sessionScope.user.role == 'ADMIN'}">
    <a href="${pageContext.request.contextPath}/admin/panel.jsp">管理后台</a>
</c:if>
```

**核心原则 (Core Principle)**:

> 前端隐藏菜单 = 用户体验优化（UX）
> 后端Filter拦截 = **真正的安全边界**
> **完整方案 = 前端隐藏 + 后端控制**
