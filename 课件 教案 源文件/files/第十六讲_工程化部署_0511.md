## 统一异常处理

###### Unified Exception Handling

**问题 (Problem)**:
- 每个Servlet都要try-catch
- 异常显示不友好（白屏、堆栈）
- 难以统一管理

**解决方案 (Solution)**:
- 全局异常过滤器
- 统一错误页面

**GlobalExceptionFilter**:
```java
@WebFilter("/*")
public class GlobalExceptionFilter implements Filter {
    public void doFilter(...) {
        try {
            chain.doFilter(req, res);
        } catch (Throwable ex) {
            // 记录日志
            ex.printStackTrace();
            // 跳转错误页
            response.sendRedirect("error.jsp");
        }
    }
}
```

## 统一日志记录

###### Unified Logging

**日志过滤器 (Logging Filter)**:

```java
@WebFilter("/*")
public class LoggingFilter implements Filter {
    public void doFilter(...) {
        long start = System.currentTimeMillis();
        String uri = request.getRequestURI();
        
        System.out.println("[ACCESS] " + uri);
        
        try {
            chain.doFilter(req, res);
        } finally {
            long cost = System.currentTimeMillis() - start;
            System.out.println("[ACCESS] " + uri + 
                             " cost=" + cost + "ms");
        }
    }
}
```

## 为什么需要配置分离？

###### Why Configuration Separation?

**硬编码的问题 (Hardcoded Problems)**:

<!-- _class: cols2_ol_ci -->

- **安全隐患** 密码在代码中  
  *Security risk*
- **无法切换** 开发/测试/生产环境  
  *Can't switch env*
- **修改麻烦** 改配置要重新编译  
  *Hard to modify*
- **团队协作** 每人配置不同  
  *Team issues*

**JNDI方案 (JNDI Solution)**:
> 将数据源配置到Tomcat容器  
> 应用通过名称查找使用  
> *Configure in container, lookup by name*

## JNDI数据源配置

###### JNDI DataSource Configuration

**Tomcat配置 (context.xml)**:

```xml
<Resource 
    name="jdbc/myDB" 
    auth="Container"
    type="javax.sql.DataSource"
    driverClassName="com.mysql.cj.jdbc.Driver"
    url="jdbc:mysql://localhost:3306/demo"
    username="root"
    password="123456"
    maxTotal="20"
    maxIdle="5"
/>
```

**应用使用 (Application Usage)**:
```java
Context ctx = new InitialContext();
DataSource ds = (DataSource) ctx.lookup("java:comp/env/jdbc/myDB");
Connection conn = ds.getConnection();
```

## WAR打包

###### WAR Packaging

**什么是WAR？** *What is WAR?*

**Web Application Archive** - Web应用归档文件

**打包方式 (Packaging Methods)**:

<!-- _class: cols2_ol_ci -->

- **IDE打包** IntelliJ/Eclipse  
  *IDE packaging*
- **Maven打包** mvn package  
  *Maven build*
- **Gradle打包** gradle war  
  *Gradle build*
- **手动打包** jar -cvf  
  *Manual packaging*

**WAR结构 (WAR Structure)**:
```
myapp.war
  ├── WEB-INF/
  │   ├── classes/    ← Java类
  │   ├── lib/        ← JAR包
  │   └── web.xml
  └── jsp/            ← JSP文件
```

## Tomcat部署

###### Tomcat Deployment

**部署步骤 (Deployment Steps)**:

```
1. 打包WAR文件
   mvn clean package → myapp.war
   ↓
2. 停止Tomcat
   bin/shutdown.bat
   ↓
3. 复制WAR到webapps/
   copy myapp.war tomcat/webapps/
   ↓
4. 启动Tomcat
   bin/startup.bat
   ↓
5. Tomcat自动解压WAR
   ↓
6. 访问测试
   http://localhost:8080/myapp/
```
