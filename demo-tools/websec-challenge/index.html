第2关：越权访问漏洞

以下 Servlet 负责管理员页面的访问控制，但存在越权访问漏洞。任何已登录用户（包括普通用户）都能访问管理页面。 请找到漏洞并编写修复代码，添加角色验证逻辑。

◆ AdminUserServlet.java
◆ User.java
◆ web.xml

@WebServlet("/admin/users") public class AdminUserServlet extends HttpServlet { protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException { // ⚠ BUG: 未检查用户角色，任何已登录用户都能访问管理页面 HttpSession session = request.getSession(); String username = (String) session.getAttribute("username"); if (username != null) { List<User> users = userDAO.findAll(); request.setAttribute("users", users); request.getRequestDispatcher("/WEB-INF/admin/users.jsp").forward(request, response); } else { response.sendRedirect("/login.jsp"); } } }
✎ 修复方案
🤖 AI助手提示
提示：检查用户Session中是否存储了角色信息，对比是否为admin角色。

建议修复思路：在验证 username != null 之后， 增加对 session.getAttribute("role") 的检查， 仅当角色为 "admin" 时才允许访问管理页面。 否则应返回403错误或重定向到无权限页面。
