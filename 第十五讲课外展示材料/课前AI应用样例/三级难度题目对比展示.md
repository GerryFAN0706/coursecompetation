## 对比表

| 维度 | 基础级（C层） | 进阶级（B层） | 挑战级（A层） |
|:---|:---|:---|:---|
| **业务场景** | 校园失物招领系统 | 实验室设备借用管理系统 | 在线考试成绩管理系统 |
| **角色数量** | 2个（发布者、普通用户） | 3个（管理员、教师、学生） | 3个 + 安全加固 |
| **核心要求** | 登录 + 登出 + Session | + 角色权限 + Filter | + BCrypt + Session安全 |
| **知识点** | Authentication, Session | + Authorization, RBAC, Filter | + Password Hash, Session Security |
| **考核项数** | 4项 | 9项 | 14项 |
| **难度跃升点** | 实现核心功能 | 理解权限分离 | 安全意识 + 工程实践 |

## "一人一题"效果说明

即使同一难度级别的学生，也会收到不同的业务场景：

| 学号 | 层级 | 收到的业务场景 |
|---|---|---|
| S001 | A层 | 在线考试成绩管理系统 |
| S005 | A层 | 校园社团活动审批系统 |
| S003 | B层 | 实验室设备借用管理系统 |
| S002 | B层 | 校园二手交易平台 |
| S006 | C层 | 校园失物招领系统 |
| S004 | C层 | 班级通知公告系统 |

> 每位学生的题目业务场景不同，但知识点考核标准相同。
> 这种设计确保评价公平性的同时，从根本上杜绝代码抄袭。
