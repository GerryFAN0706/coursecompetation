# C组任务卡：视图一致性组（View Consistency）

## 你的任务

审查系统的 **JSP 页面菜单显示逻辑**，判断前端展示是否与后端权限控制一致。

## 审查文件

1. `index.jsp` — 首页（重点关注菜单导航区域）

## 关注问题

1. 菜单是否根据 `sessionScope.user` 正确显示登录状态？
2. "管理后台"入口是否根据角色（ADMIN/USER）正确显示或隐藏？
3. 前端菜单隐藏是否与服务端权限控制一致？
4. 是否存在"前端隐藏了入口，但直接输入 URL 仍可访问"的风险？

## AI 审查提示词

将以下内容复制到 AI 工具中，并附上 `index.jsp` 的代码：

```
请审查这个 Java Web JSP 页面中的菜单显示逻辑，重点检查：

1. 是否根据 sessionScope.user 正确显示登录状态
2. 是否根据角色正确显示"管理后台"入口
3. 前端菜单隐藏是否与服务端权限控制一致
4. 是否存在"前端隐藏了入口，但直接输入 URL 仍可访问"的风险

请按以下格式输出：
- 问题位置
- 风险说明
- 对应知识点（View Rendering / Session / Authorization）
- 修改建议
- 建议验证方式
```

## 汇报要求

准备回答以下三个问题（1-2分钟）：

1. AI 发现了什么问题？
2. 对应哪些课程知识点？
3. 哪个建议最值得采纳？

## 对应知识点

- **View Rendering**（视图渲染）：页面展示逻辑
- **Session**（会话）：如何从会话中读取用户信息
- **Authorization**（授权）：前端隐藏菜单 ≠ 真正的安全控制
