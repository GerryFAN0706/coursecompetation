# 推广与复用

## 可复制教学模式

```text
AI个性化任务设计
 + AI辅助知识点对照与代码审查
 + 数据驱动多维评价反馈
 = 课前-课中-课后教学闭环
```

## 最低实施条件与推荐配置

| 条件 | 最低要求 | 推荐配置 |
|---|---|---|
| AI工具 | 任一可用生成式AI工具 | IDE内嵌AI + API工具链 |
| 学生设备 | 可联网电脑 | 人手一机开发环境 |
| 教师能力 | 基本提示词使用 | 能设计提示词模板和评价规则 |
| 班级规模 | 10-60人 | 30-40人 |

## 适配课程范围

1. Java Web应用开发（已验证）
2. Python程序设计（可迁移）
3. 数据结构与算法（代码审查+测试验证）
4. 数据库课程实验（SQL审查+分层任务）
5. 软件工程实践（项目审查+多维评价）

## 输出成果与复用资产

1. 可复用提示词模板库
2. 分层任务生成流程与样例
3. 测试清单与反馈模板
4. 课堂实录脚本与执行方案
5. 可公开访问的课程展示网站

## 三步迁移法（给其他课程）

1. 先选一个高频痛点任务，建立“问题版案例”。
2. 再接入 AI 审查与测试清单，形成课堂验证闭环。
3. 最后接入数据记录与反馈模板，形成课后干预机制。

## 实施建议

- 优先从一个“可验证问题”切入，不追求全课程一次性改造。
- 先做 1 个核心案例闭环，再扩展到多讲次和多任务场景。
- 所有关键指标都绑定证据来源和统计口径，确保可评审、可复核。
