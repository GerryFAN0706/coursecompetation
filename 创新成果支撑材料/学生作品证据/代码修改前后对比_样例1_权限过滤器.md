## 修改前代码（学生初始提交）

```java
@WebFilter("/admin/*")
public class RoleBasedAuthFilter implements Filter {
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        HttpSession session = request.getSession(false);

        // 只检查了是否登录，没有检查角色
        boolean loggedIn = session != null
                && session.getAttribute("user") != null;

        if (loggedIn) {
            chain.doFilter(req, res);  // 所有登录用户都能进入
        } else {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }
}
```

### 存在的问题
1. 只判断`user`是否存在（认证），未判断角色是否为ADMIN（授权）
2. 普通用户（USER角色）登录后也能访问`/admin/*`路径
3. 混淆了Authentication和Authorization的概念

## 修改后代码（学生最终提交）

```java
@WebFilter("/admin/*")
public class RoleBasedAuthFilter implements Filter {
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        HttpSession session = request.getSession(false);

        if (session == null) {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
            return;
        }

        User user = (User) session.getAttribute("user");
        if (user != null && "ADMIN".equals(user.getRole())) {
            chain.doFilter(req, res);
        } else if (user != null) {
            response.sendError(HttpServletResponse.SC_FORBIDDEN);
        } else {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }
}
```

### 改进点
1. 从Session中获取User对象，检查`user.getRole()`是否为ADMIN
2. 区分三种情况：未登录→重定向、已登录非ADMIN→403、ADMIN→放行
3. 正确实现了Authorization逻辑，符合RBAC模型
