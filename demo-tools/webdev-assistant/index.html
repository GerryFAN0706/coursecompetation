老师好，我想问一下Session和Cookie到底有什么区别？什么时候该用哪个？
14:22
W
这是个很好的问题！在 第5讲 会话管理 中我们详细讲解了这个知识点，我帮你梳理一下核心区别：

1. 存储位置不同
• Cookie 存储在客户端浏览器中
• Session 存储在服务器端（默认在内存中）

2. 安全性差异
• Cookie 可被用户查看和篡改，安全性较低
• Session 数据在服务器端，相对安全

3. 生命周期
• Cookie 可设置过期时间，持久存储
• Session 默认随浏览器关闭而失效（Tomcat默认30分钟超时）

4. 典型使用场景
• Cookie：记住用户名、语言偏好等非敏感信息
• Session：登录状态、购物车、验证码等敏感数据

// 第5讲 代码示例：Session存储登录信息 HttpSession session = request.getSession(); session.setAttribute("currentUser", user); // Cookie记住用户偏好 Cookie cookie = new Cookie("theme", "dark"); cookie.setMaxAge(7 * 24 * 3600); response.addCookie(cookie);
记忆口诀：敏感数据用Session，偏好记忆用Cookie。面试中也经常考到这个对比哦！

李
那Session的JSESSIONID和Cookie是什么关系？
14:23
W
问得好！这正是 第5讲 的一个重点，也是很多同学容易混淆的地方：

Session 依赖 Cookie 来工作！具体流程是：

• 服务器创建Session后，生成一个唯一的 JSESSIONID
• 这个ID通过 Cookie 发送给浏览器（名为JSESSIONID的Cookie）
• 浏览器后续请求自动携带这个Cookie
• 服务器根据JSESSIONID找到对应的Session对象

所以本质上：Session是机制，Cookie是载体。如果浏览器禁用了Cookie，可以用URL重写方式传递JSESSIONID。

这部分在期末考试中占比较高，建议结合 第5讲 实验 用Servlet代码实际调试一下！
