# 学生代码修改前后对比

> 用途：视频中 9:00-10:00 段落展示
> 模拟一位B层学生收到AI反馈后的二次迭代修改

---

## 修改1：RoleBasedAuthFilter — 增加角色判断

### 修改前

```java
@WebFilter("/admin/*")
public class RoleBasedAuthFilter implements Filter {
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        HttpSession session = request.getSession(false);

        // 问题：只判断了用户是否存在，未检查角色
        if (session != null && session.getAttribute("user") != null) {
            chain.doFilter(req, res);
        } else {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }
}
```

### 修改后

```java
@WebFilter("/admin/*")
public class RoleBasedAuthFilter implements Filter {
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        HttpSession session = request.getSession(false);

        if (session == null) {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
            return;
        }

        User user = (User) session.getAttribute("user");
        if (user != null && "ADMIN".equals(user.getRole())) {
            chain.doFilter(req, res);
        } else if (user != null) {
            response.sendError(HttpServletResponse.SC_FORBIDDEN);
        } else {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }
}
```

**修改要点：**
- 从Session中取出User对象，检查 `getRole()` 是否为ADMIN
- 区分三种情况：未登录→重定向、已登录无权限→403、管理员→放行
- 对应知识点：Authorization、RBAC

## 修改2：LogoutServlet — 彻底销毁Session

### 修改前

```java
@WebServlet("/logout")
public class LogoutServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
        HttpSession session = req.getSession(false);
        if (session != null) {
            // 问题：只删除了user属性，Session本身仍然存在
            session.removeAttribute("user");
        }
        resp.sendRedirect(req.getContextPath() + "/login.jsp");
    }
}
```

### 修改后

```java
@WebServlet("/logout")
public class LogoutServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
        HttpSession session = req.getSession(false);
        if (session != null) {
            session.invalidate();
        }
        resp.sendRedirect(req.getContextPath() + "/login.jsp");
    }
}
```

**修改要点：**
- `removeAttribute` → `invalidate`
- 一行代码的变化，但意义完全不同：前者只删一个属性，后者销毁整个Session
- 对应知识点：Session生命周期管理

## 修改3：index.jsp — 按角色显示菜单

### 修改前

```jsp
<c:if test="${sessionScope.user != null}">
    <a href="${pageContext.request.contextPath}/admin/manage.jsp">设备管理</a>
    <a href="${pageContext.request.contextPath}/logout">退出</a>
</c:if>
```

### 修改后

```jsp
<c:if test="${sessionScope.user != null}">
    <c:if test="${sessionScope.user.role == 'ADMIN'}">
        <a href="${pageContext.request.contextPath}/admin/manage.jsp">设备管理</a>
    </c:if>
    <c:if test="${sessionScope.user.role == 'TEACHER' || sessionScope.user.role == 'ADMIN'}">
        <a href="${pageContext.request.contextPath}/teacher/approve.jsp">审批借用</a>
    </c:if>
    <a href="${pageContext.request.contextPath}/student/myborrows.jsp">我的借用</a>
    <a href="${pageContext.request.contextPath}/logout">退出</a>
</c:if>
```

**修改要点：**
- 菜单项按角色条件判断显示
- 管理员看到全部菜单，教师看到审批+借用，学生只看到借用
- 对应知识点：视图渲染与权限一致性

## 对比总结

| 修改点 | 修改前的问题 | 修改后的效果 | 对应知识点 |
|:---|:---|:---|:---|
| Filter | 只判断登录，不判断角色 | 区分角色，非管理员返回403 | Authorization, RBAC |
| LogoutServlet | removeAttribute，Session残留 | invalidate，彻底销毁 | Session生命周期 |
| index.jsp | 登录即显示所有菜单 | 按角色条件显示 | 视图与权限一致性 |
