## Controller直连数据库的问题

###### Problems of Direct Database Access

**当前做法：Servlet中混杂一切** *Current: Everything in Servlet*

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    protected void doPost(...) {
        String username = request.getParameter("username");
        
        // Servlet直接连接数据库！
        Connection conn = DBUtil.getConnection();
        PreparedStatement ps = conn.prepareStatement(
            "SELECT * FROM users WHERE username=?");
        ps.setString(1, username);
        ResultSet rs = ps.executeQuery();
        // ... 处理结果、关闭资源、转发JSP ...
        // 业务逻辑、数据访问、流程控制混在一起！
    }
}
```

## 问题分析

###### Problem Analysis

**混乱的职责 (Mixed Responsibilities)**:

<!-- _class: cols2_ol_ci -->

- **问题1** Controller管太多  
  *Controller does too much*
- **问题2** JDBC代码分散  
  *JDBC code scattered*
- **问题3** 难以测试  
  *Hard to test*
- **问题4** 代码重复  
  *Code duplication*
- **问题5** 难以维护  
  *Hard to maintain*
- **问题6** 无法复用  
  *Not reusable*

**需求 (Need)**:
> 将数据访问从业务逻辑中**分离**出来！  
> *Separate data access from business logic!*

## 理想的分层

###### Ideal Layering

**各层只做自己的事 (Each layer does its own job)**:

```
Controller层: 只管接收请求、调用Service、转发View
  ↓
Service层: 只管业务逻辑编排、事务控制
  ↓
DAO层: 只管数据库访问、SQL执行
  ↓
Database: 数据存储
```

**原则**: 关注点分离、职责单一  
*Principle: Separation of concerns, single responsibility*

## 包结构规范

###### Package Structure

**标准包结构 (Standard Structure)**:

```
src/com/tyust/
  ├── controller/     ← Servlet控制器
  │   └── LoginServlet.java
  ├── service/        ← 业务逻辑层
  │   └── UserService.java
  ├── dao/            ← 数据访问层
  │   ├── UserDao.java         (接口)
  │   └── impl/
  │       └── JdbcUserDao.java (实现)
  ├── model/          ← 实体模型
  │   └── User.java
  └── util/           ← 工具类
      └── DBUtil.java
```

**命名规范**: XxxServlet, XxxService, XxxDao

## 依赖方向

###### Dependency Direction

**单向依赖原则 (Unidirectional Dependency)**:

```
Controller → Service → DAO → Database
```

**不允许 (Not Allowed)**:
- ❌ DAO不能调用Service
- ❌ Service不能依赖Controller
- ❌ Model不能依赖任何层

**原则**: 上层依赖下层，下层不知道上层  
*Principle: Upper depends on lower, not vice versa*

## DAO模式简介

###### DAO Pattern Introduction

<!-- _class: bq-blue -->

> **什么是DAO？ (What is DAO?)**
>
> **Data Access Object** - 数据访问对象  
> 封装所有数据库操作，提供统一访问接口  
> *Encapsulates all database operations*

**核心思想 (Core Idea)**:
- 🔌 接口抽象 *Interface abstraction*
- 🛡️ 屏蔽细节 *Hide details*
- ♻️ 提高复用 *Improve reusability*

## DAO接口设计

###### DAO Interface Design

**定义数据访问接口 (Define interface)**:

```java
// UserDao.java - 接口
public interface UserDao {
    User findByUsername(String username);
    User findById(Long id);
    List<User> findAll();
    int insert(User user);
    int update(User user);
    int delete(Long id);
}
```

**特点 (Features)**:
- 方法名语义清晰 *Clear method names*
- 只定义行为，不含实现 *Behavior only*
- 便于切换实现（JDBC/MyBatis/Hibernate）

## DAO实现类

###### DAO Implementation

**JDBC实现 (JDBC Implementation)**:

```java
// JdbcUserDao.java
public class JdbcUserDao implements UserDao {
    
    @Override
    public User findByUsername(String username) {
        String sql = "SELECT * FROM users WHERE username=?";
        Connection conn = null;
        PreparedStatement ps = null;
        ResultSet rs = null;
        
        try {
            conn = DBUtil.getConnection();
            ps = conn.prepareStatement(sql);
            ps.setString(1, username);
            rs = ps.executeQuery();
            
            if (rs.next()) {
                User user = new User();
                user.setId(rs.getLong("id"));
                user.setUsername(rs.getString("username"));
                return user;
            }
        } catch (SQLException e) {
            e.printStackTrace();
        } finally {
            DBUtil.close(conn, ps, rs);
        }
        return null;
    }
}
```

## Service层编排

###### Service Layer Orchestration

**业务逻辑层 (Business Logic Layer)**:

```java
// UserService.java
public class UserService {
    private UserDao dao = new JdbcUserDao();
    
    public User login(String username, String password) {
        // 1. 查询用户 (Query user)
        User user = dao.findByUsername(username);
        
        // 2. 验证密码 (Verify password)
        if (user != null && user.getPassword().equals(password)) {
            return user;
        }
        
        return null;
    }
}
```

**职责 (Responsibilities)**:
- 调用DAO获取数据 *Call DAO*
- 执行业务验证 *Business validation*
- 返回处理结果 *Return result*

## Controller层精简

###### Controller Layer Simplified

**Servlet只负责协调 (Servlet only coordinates)**:

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    private UserService service = new UserService();
    
    protected void doPost(...) {
        // 1. 获取参数 (Get parameters)
        String username = request.getParameter("username");
        String password = request.getParameter("password");
        
        // 2. 调用Service (Call service)
        User user = service.login(username, password);
        
        // 3. 转发View (Forward to view)
        if (user != null) {
            request.setAttribute("user", user);
            request.getRequestDispatcher("/welcome.jsp").forward(...);
        } else {
            request.setAttribute("error", "登录失败");
            request.getRequestDispatcher("/login.jsp").forward(...);
        }
    }
}
```

## 完整数据流

###### Complete Data Flow

**三层协作流程 (Three-Tier Collaboration)**:

```
1. 用户访问 /login → LoginServlet
   ↓
2. Controller获取参数 → 调用Service
   ↓
3. Service.login() → 调用DAO
   ↓
4. DAO.findByUsername() → JDBC查询数据库
   ↓
5. 封装User对象 → 返回给Service
   ↓
6. Service验证密码 → 返回给Controller
   ↓
7. Controller存入request → forward到JSP
   ↓
8. JSP用EL显示 ${user} → 返回HTML
```

## DAO接口的优势

###### Advantages of DAO Interface

**为什么使用接口？** *Why use interface?*

<!-- _class: cols2_ol_ci -->

- **抽象访问** 隐藏实现细节  
  *Abstract access*
- **易于切换** JDBC→MyBatis→Hibernate  
  *Easy to switch*
- **便于测试** Mock实现  
  *Easy to test*
- **规范统一** 团队协作  
  *Unified standard*

**示例 (Example)**:
```java
// 接口不变，实现可替换
UserDao dao = new JdbcUserDao();      // JDBC实现
UserDao dao = new MyBatisUserDao();   // MyBatis实现
```

## 最佳实践

###### Best Practices

<!-- _class: bq-green -->

> **分层开发建议 (Layered Development Recommendations)**
>
> - ✅ 自底向上开发：Model→DAO→Service→Controller  
>   *Bottom-up: Model→DAO→Service→Controller*
>
> - ✅ DAO只返回实体对象，不返回ResultSet  
>   *DAO returns entity objects, not ResultSet*
>
> - ✅ Service层处理事务和异常转换  
>   *Service handles transactions and exceptions*
>
> - ✅ Controller层保持轻量，只做协调  
>   *Keep Controller lightweight*
