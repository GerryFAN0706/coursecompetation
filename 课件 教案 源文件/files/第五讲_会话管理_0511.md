## HTTP的无状态性

###### HTTP's Stateless Nature

<!-- _class: bq-blue -->

> **什么是无状态？ (What is Stateless?)**
>
> HTTP协议本身不保留任何请求状态信息  
> 每个请求都是独立的，服务器不记得上一个请求  
> *Each request is independent, server doesn't remember previous requests*

**形象比喻 (Analogy)**:
> 就像你每次去银行办事，都得重新出示身份证  
> 银行柜员不会记得你刚才来过  
> *Like showing your ID every time at the bank*

**问题 (Problem)**:
- 😫 每次请求都要重新验证身份
- 😫 无法实现"登录状态"
- 😫 无法实现"购物车"
- 😫 用户体验极差

## 我们需要什么？

###### What Do We Need?

**理想方案 (Ideal Solution)**:

用户**一次登录**，在整个访问期间都保持登录状态  
*Login once, stay logged in throughout the visit*

**需求 (Requirements)**:

<!-- _class: cols2_ol_ci -->

- **记住用户** 跨多个请求  
  *Remember user*
- **存储状态** 持续整个会话  
  *Store state*
- **自动关联** 识别同一用户  
  *Auto associate*
- **安全可靠** 防止冒充  
  *Secure*

**解决方案 (Solution)**: `HttpSession` 会话管理！

## HttpSession：服务器的记忆

###### HttpSession: Server's Memory

<!-- _class: bq-blue -->

> **什么是HttpSession？ (What is HttpSession?)**
>
> 服务器为每个用户分配的"专属储物柜"  
> 存储该用户在整个访问期间的状态数据  
> *A "personal locker" assigned by server for each user*

**储物柜比喻 (Locker Analogy)**:

```
用户首次访问 → 服务器分配储物柜 → 给用户钥匙
  ↓
用户再次访问 → 带着钥匙 → 服务器找到对应储物柜
  ↓
用户可以存取数据，贯穿整个访问过程
```

## JSESSIONID：会话的钥匙

###### JSESSIONID: The Session Key

**什么是JSESSIONID？** *What is JSESSIONID?*

一个由服务器生成的唯一ID，存储在Cookie中  
*A unique ID generated by server, stored in Cookie*

**工作机制 (How it works)**:

```
1. 首次访问
   服务器创建Session → 生成ID: "ABC123XYZ"
   ↓
   返回Cookie: JSESSIONID=ABC123XYZ
   ↓
   浏览器保存Cookie

2. 再次访问
   浏览器自动携带Cookie: JSESSIONID=ABC123XYZ
   ↓
   服务器根据ID找到对应Session
   ↓
   获取用户数据
```

## 访问控制：登录检查

###### Access Control: Login Check

**任何需要登录的页面都要检查 (Check login on protected pages)**:

```java
@WebServlet("/profile")
public class ProfileServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, ...) {
        // 检查登录状态 (Check login status)
        HttpSession session = request.getSession(false);  // 不创建
        
        if (session == null || 
            session.getAttribute("loggedInAgent") == null) {
            // 未登录，跳转到登录页 (Not logged in, redirect)
            response.sendRedirect("login.jsp");
            return;
        }
        
        // 已登录，继续处理 (Logged in, continue)
        AgentBean agent = 
            (AgentBean) session.getAttribute("loggedInAgent");
        // ...
    }
}
```

## 实现登出功能

###### Implement Logout

**LogoutServlet**:

```java
@WebServlet("/logout")
public class LogoutServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request,
                         HttpServletResponse response) {
        // 1. 获取Session (Get session)
        HttpSession session = request.getSession(false);
        
        // 2. 销毁Session (Invalidate session)
        if (session != null) {
            session.invalidate();  // 核心方法！
        }
        
        // 3. 跳转到登录页 (Redirect to login)
        response.sendRedirect("login.jsp");
    }
}
```

**效果 (Effect)**:
- 销毁整个Session对象
- 清除所有存储的数据
- 用户需要重新登录
