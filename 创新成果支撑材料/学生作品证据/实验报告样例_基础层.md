## 一、实验目标

将课堂示例工程中的`PasswordUtil`从SHA-256升级为BCrypt算法，理解两种算法在安全性上的差异。

## 二、实验环境

- JDK 8、Maven 3.8、Tomcat 9
- 新增依赖：`jBCrypt 0.4`

## 三、实验步骤

### 3.1 添加BCrypt依赖

```xml
<dependency>
    <groupId>org.mindrot</groupId>
    <artifactId>jbcrypt</artifactId>
    <version>0.4</version>
</dependency>
```

### 3.2 修改PasswordUtil

```java
public final class PasswordUtil {
    public static String hashPassword(String password) {
        return BCrypt.hashpw(password, BCrypt.gensalt(12));
    }

    public static boolean matches(String rawPassword, String hashedPassword) {
        return BCrypt.checkpw(rawPassword, hashedPassword);
    }
}
```

### 3.3 更新UserDao中的预置用户密码

```java
static {
    Map<String, User> temp = new HashMap<>();
    temp.put("admin", new User("admin",
        PasswordUtil.hashPassword("admin123"), "ADMIN", "课程管理员"));
    temp.put("tom", new User("tom",
        PasswordUtil.hashPassword("tom123"), "USER", "普通学生 Tom"));
    USERS = Collections.unmodifiableMap(temp);
}
```

## 五、SHA-256 vs BCrypt对比分析

| 维度 | SHA-256 | BCrypt |
|:---|:---|:---|
| 算法类型 | 消息摘要（快速） | 自适应哈希（慢速） |
| 盐值 | 需要手动加盐 | 自动内置盐值 |
| 计算速度 | 极快（GPU可加速） | 故意慢（防暴力破解） |
| 彩虹表攻击 | 无盐时易受攻击 | 每次哈希不同，无法预计算 |
| 适用场景 | 数据完整性校验 | **密码存储** |

## 六、实验总结

通过本次实验，我理解了为什么课堂上老师说"SHA-256用于演示原理，生产环境应使用BCrypt"。BCrypt的关键优势是**故意设计得慢**——这对正常登录影响微乎其微（约100ms），但对暴力破解攻击来说，每次尝试都要付出100ms的代价，使得暴力破解在实际中不可行。
