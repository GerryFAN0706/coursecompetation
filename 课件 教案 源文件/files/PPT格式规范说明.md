## 🚨 常见错误汇总（重要！）

根据第一讲和第二讲的优化实践，总结以下**最容易犯的错误**：

### ❌ 错误1：空白页问题（最严重）

### 问题根源
由于在YAML配置中设置了 `headingDivider: [2,3]`，每个 `##` 和 `###` 标题会**自动创建新页面**。

但代码中在许多 `##` 标题前又手动添加了 `---` 分页符，导致：
- 手动 `---` 创建了**空白页**
- `##` 自动分页又创建了**内容页**
- 结果：每个位置都多了一个空白页

**问题表现**：
- 每个页面后出现空白蓝色页面
- 过渡页之后有空白页
- 总页数异常增多（本应20页变成40页）

**根本原因**：
```yaml
headingDivider: [2,3]  # 让##和###自动分页
```
但又在 `##` 前手动添加 `---`，导致双重分页！

**解决方案** ✅:
```markdown
# 错误写法 ❌
> 内容结束

---                     ← 多余！会产生空白页

## 下一页标题            ← ##已经自动分页

# 正确写法 ✅  
> 内容结束

## 下一页标题            ← 只靠##自动分页即可
```

**删除位置**：
- ❌ 任何 `##` 标题前的 `---`
- ❌ 任何 `###` 标题前的 `---`
- ❌ `</div>` 后的 `---`
- ❌ 引用盒子后的 `---`

**保留位置**：
- ✅ 只在YAML结束处保留 `---`
- ✅ 第一个过渡页需要时保留（可选）

### ❌ 错误2：过渡页使用两个`##`标题

**错误写法**：
```markdown
## 第一部分：问题引入
## Part I: Introduction    ← 第二个##触发自动分页！产生空白页
```

**正确写法** ✅:
```markdown
## 第一部分：问题引入

###### Part I: Introduction  ← 使用######不会分页
```

### ❌ 错误3：class指令位置错误

**错误写法**：
```markdown
<!-- _class: trans -->        ← class在标题前
## 第一部分：问题引入
```

**正确写法** ✅:
```markdown
## 第一部分：问题引入

###### Part I: Introduction

<!-- _class: lastpage -->      ← class在标题后
<!-- _footer: "" -->
<!-- _paginate: "" -->
```

### ❌ 错误4：分栏内部使用`###`标题

**错误写法**：
```markdown
<div class="ldiv">

### 开发侧 (Development)    ← ###会触发分页，内容分离！
```

**正确写法** ✅:
```markdown
<div class="ldiv">

#### 开发侧 (Development)    ← 使用####不会分页
```

### ❌ 错误5：双语格式不统一

**错误写法**：
```markdown
## 核心概念 (Core Concept)              ← 混在一起
## HTTP协议：Web的通用语 (HTTP: ...)     ← 混在一起
```

**正确写法** ✅:
```markdown
## 核心概念

###### Core Concept              ← 中英文分开，更清晰
```

### ❌ 错误6：Logo路径问题

**错误写法**：
```markdown
<!-- _footer: ![h:100](./logo.png) -->  ← 多余的 ./
```

**正确写法** ✅:
```markdown
<!-- _footer: ![h:100](logo.png) -->    ← 直接文件名
```

### ❌ 错误7：footer缺少反斜杠

**错误写法**：
```yaml
footer: *Java Web应用开发* *学院*     ← 缺少 \ 
```

**正确写法** ✅:
```yaml
footer: \ *Java Web应用开发* *学院*   ← 必须有 \ 开头
```

### ❌ 错误8：内容过长页面使用彩色背景

**错误写法**：
```markdown
<!-- _class: bq-blue -->      ← 彩色背景

## 长内容页面

> 这里有10行以上的内容...
> 大段文字...
> 多个段落...
```

**正确写法** ✅:
```markdown
## 长内容页面              ← 白色背景

**要点**:
- 内容1
- 内容2
...
```

## ✅ 正确的分页逻辑（重要！）

### YAML配置
```yaml
headingDivider: [2,3]  # ##和###自动分页
```

### 完整的页面结构示例
```markdown
---
marp: true
headingDivider: [2,3]
---

<!-- 封面页 -->
# 标题

## 本讲概要              ← ##自动分页→第2页

## 教学目标              ← ##自动分页→第3页

## 教学目标              ← ##自动分页→第4页

## 第一部分：问题引入     ← ##自动分页→第5页（过渡页）

###### Part I: Introduction

<!-- _class: lastpage -->  ← class在标题后

## 问题引入内容          ← ##自动分页→第6页

### 子标题              ← ###自动分页→第7页
```

**关键规则**：
1. ✅ 信任 `headingDivider` 自动分页
2. ✅ 不要在 `##` 和 `###` 前添加 `---`
3. ✅ 英文副标题用 `######`（不触发分页）
4. ✅ class指令放在标题内容之后

## 📐 标题层级使用规范

### 标题级别与分页关系

| 标题 | 语法 | 是否分页? | 适用场景 | 示例 |
|------|------|----------|---------|------|
| 一级 | `#` | ✅ 是 | 封面主标题 | `# 课程标题` |
| 二级 | `##` | ✅ 是 | 页面主标题 | `## 核心概念` |
| 三级 | `###` | ✅ 是 | 小节标题 | `### 请求三要素` |
| 四级 | `####` | ❌ 否 | 分栏内标题 | `#### 开发侧` |
| 五级 | `#####` | ❌ 否 | 段落标题 | - |
| 六级 | `######` | ❌ 否 | **英文副标题** | `###### Part I` |

**最佳实践**：
- ✅ 页面中文标题用 `##`
- ✅ 英文副标题用 `######`
- ✅ 分栏内部标题用 `####`
- ❌ 不要在过渡页用两个 `##`

## 📐 双语格式规范

### 统一的双语标题格式

**推荐格式**：
```markdown
## 中文标题

###### English Subtitle
```

**❌ 不推荐**：
```markdown
## 中文标题 (English)           ← 太长，拥挤
## 中文标题
## English Subtitle            ← 会产生空白页
```

### 正文双语格式

**简短说明**：
```markdown
我们需要一个服务器。  
*We need a server.*
```

**列表项**：
```markdown
- 接收请求 (Receive requests)
- 处理数据 (Process data)
```

**引用盒子内**：
```markdown
> 这是核心概念  
> *(This is the core concept)*
```

## 📐 格式规范

### 1. 文字多的页面：避免彩色背景 🎨

**规则**: 当页面包含大量文字内容时，**不使用彩色背景**的引用盒子样式。

#### ❌ 不推荐（文字多+彩色背景）
```markdown
<!-- _class: bq-blue -->

## 核心概念

> 这里有很长的文字说明...
> 包含多个段落...
> 大量技术细节...
> 代码示例...
> 表格数据...
```

**问题**：
- 彩色背景降低可读性
- 文字与背景对比度不足
- 投影时效果差

#### ✅ 推荐（文字多用白底）
```markdown
## 核心概念

这里有很长的文字说明...
包含多个段落...
大量技术细节...

**要点**:
- 项目1
- 项目2
```

**优势**：
- 白色背景，文字清晰
- 易于阅读和投影
- 专业简洁

---

### 2. 背景色与文字颜色搭配 🎨

#### 使用彩色背景的场景
**仅在以下情况使用彩色引用盒子**：

✅ **简短概念定义**（1-2段）
```markdown
<!-- _class: bq-blue -->

## 核心概念

> **HTTP协议 (HTTP Protocol)**
>
> 浏览器和服务器通信的"通用语言"。
```

✅ **重要提示框**（强调用）
```markdown
<!-- _class: bq-red -->

## 注意事项

> ⚠️ **重要提醒**
>
> 部署前务必检查端口配置。
```

✅ **课程思政内容**（特殊标识）
```markdown
<!-- _class: bq-purple -->

## 课程思政

> **数字中国战略**
>
> 学习Web技术是参与国家战略的基础。
```

#### 颜色使用指南

| 样式 | 颜色 | 适用场景 | 最大文字量 |
|------|------|---------|-----------|
| `bq-blue` | 蓝色 | 概念定义、技术说明 | 3-4行 |
| `bq-green` | 绿色 | 最佳实践、正面提示 | 3-4行 |
| `bq-red` | 红色 | 警告、注意事项 | 2-3行 |
| `bq-purple` | 紫色 | 课程思政内容 | 4-5行 |
| `bq-black` | 黑色 | 严肃内容、规范 | 3-4行 |

**文字超过5行？→ 使用白色背景！**

### 3. 删除个人信息 🔒

**规则**: 删除所有个人身份信息和联系方式。

#### 需要删除的内容
- ❌ 教师姓名（封面页、页脚）
- ❌ 联系邮箱
- ❌ 微信号
- ❌ 电话号码
- ❌ 个人照片

#### 保留的内容
- ✅ 单位名称（学校）
- ✅ 学院名称
- ✅ 课程信息
- ✅ 学期信息

#### 修改示例

**封面页 - 修改前**：
```markdown
**主讲教师**: XXX
**授课方式**: 双语教学 (Bilingual Teaching)
**单位**: XXX大学·XXX学院
**时间**: 2025年秋季学期
```

**封面页 - 修改后**：
```markdown
**课程名称**: Java Web 应用开发
**授课方式**: 双语教学 (Bilingual Teaching)
**开课单位**: XXX学院
**学期**: 2025年秋季
```

**页脚 - 修改前**：
```yaml
footer: \ *XXX* *Java Web应用开发* *XXX大学*
```

**页脚 - 修改后**：
```yaml
footer: *Java Web应用开发 (Java Web Application Development)* *XXX学院*
```

**结束页 - 删除**：
```markdown
<!-- 删除整个联系信息区域 -->
- <i class="fa-solid fa-envelope"></i>
  - xxx@example.edu.cn
```

### 4. 增大文字尺寸 📝

**规则**: 提升整体文字大小，确保后排学生可见。

#### 方法一：在YAML中全局调整（推荐）

在文件开头的YAML区域添加：

```yaml
---
marp: true
size: 16:9
theme: am_blue
paginate: true
headingDivider: [2,3]
footer: *Java Web应用开发* *XXX学院*
style: |
  section {
    font-size: 28px;  /* 原默认25px → 28px */
  }
  h2, h3 {
    font-size: 42px;  /* 原默认38px → 42px */
  }
  h4, h5 {
    font-size: 34px;  /* 原默认30px → 34px */
  }
  li {
    font-size: 26px;  /* 列表项 */
    line-height: 1.6;
  }
  code {
    font-size: 24px;  /* 代码字体 */
  }
---
```

#### 方法二：使用页面级样式调整

对特定页面使用：
```markdown
<!-- _class: largetext -->  ## 文字会自动放大15%
<!-- _class: hugetext -->   ## 文字会自动放大30%
```

#### 文字大小对照表

| 元素 | 原大小 | 推荐大小 | 增幅 |
|------|--------|---------|------|
| 正文 | 25px | **28px** | +12% |
| 二/三级标题 | 38px | **42px** | +11% |
| 四/五级标题 | 30px | **34px** | +13% |
| 列表项 | 25px | **26px** | +4% |
| 代码 | 22px | **24px** | +9% |

**特殊场景**：
- 如果内容太多挤不下 → 使用 `<!-- _class: smalltext -->`（90%大小）
- 如果是关键概念页 → 使用 `<!-- _class: largetext -->`（115%大小）

## 🎯 Awesome-Marp特定规范

### 过渡页设计规范

**标准过渡页格式**：
```markdown
## 第一部分：问题引入

###### Part I: Introduction

<!-- _class: lastpage -->     ← 使用lastpage样式效果更好
<!-- _footer: "" -->
<!-- _paginate: "" -->
```

**效果**：
- 蓝色渐变背景
- 白色大字标题
- 中英文分层显示

### 封面页Logo设置

**推荐使用 cover_d 样式**：
```markdown
<!-- _class: cover_d -->
<!-- _header: "" -->
<!-- _footer: ![h:100](logo.png) -->  ← logo在右下角
<!-- _paginate: "" -->
```

**Logo位置微调**（在YAML的style中）：
```yaml
style: |
  section.cover_d footer {
    left: 75%;      # 70-85%，越大越靠右
    bottom: 25px;   # 10-50px，越大越靠上
  }
```

### 内容精简原则

**每页建议内容量**：
- 📄 纯文字页面：10-15行
- 💻 代码页面：15-20行代码
- 📊 列表页面：6-8个要点
- 🎨 引用盒子：3-5行

**超过建议量的处理**：
1. 拆分成两页（使用 `##` 自动分页）
2. 使用分栏布局（`cols-2`）
3. 精简文字，提取关键点

## 🛠️ 常见问题快速修复

### 问题排查流程

**如果出现空白页**：
1. 搜索 `^---$` 检查所有分页符
2. 删除 `##` 和 `###` 前的所有 `---`
3. 检查过渡页是否使用了两个 `##`

**如果过渡页不显示**：
1. 确认 `<!-- _class: lastpage -->` 在标题之后
2. 确认有 `<!-- _footer: "" -->` 隐藏footer
3. 确认有 `<!-- _paginate: "" -->` 隐藏页码

**如果分栏内容分页**：
1. 检查分栏内是否使用了 `###` 标题
2. 改为 `####` 或 `#####` 标题

**如果logo不显示**：
1. 检查路径：使用 `logo.png` 而不是 `./logo.png`
2. 确认文件存在于 `files/` 目录
3. 检查 `cover_d footer` 样式是否在YAML的style中

**如果footer不显示**：
1. 确认footer开头有 `\ ` (反斜杠+空格)
2. 检查YAML格式是否正确
