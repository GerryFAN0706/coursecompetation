> **难度层级**：进阶（Advanced）
> **任务**：增加MANAGER角色，实现三级权限（USER/MANAGER/ADMIN）

---

## 一、实验目标

在现有双角色（USER/ADMIN）的基础上，新增MANAGER角色，实现三级权限控制：
- USER：仅访问首页和个人页面
- MANAGER：可访问`/report/*`报表页面
- ADMIN：可访问`/admin/*`后台管理

## 二、设计思路

### 2.1 角色与路径映射

| 角色 | 可访问路径 | 不可访问路径 |
|:---|:---|:---|
| USER | `/`, `/index.jsp`, `/user/*` | `/report/*`, `/admin/*` |
| MANAGER | `/`, `/index.jsp`, `/user/*`, `/report/*` | `/admin/*` |
| ADMIN | 所有路径 | 无 |

### 2.2 新增文件

- `ReportAuthFilter.java` — 拦截`/report/*`，只允许MANAGER和ADMIN
- `report/dashboard.jsp` — 报表页面
- 修改`UserDao` — 新增MANAGER预置用户
- 修改`index.jsp` — 按角色显示三种菜单

## 三、核心代码

### 3.1 ReportAuthFilter

```java
@WebFilter("/report/*")
public class ReportAuthFilter implements Filter {
    private static final Set<String> ALLOWED_ROLES =
        new HashSet<>(Arrays.asList("MANAGER", "ADMIN"));

    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        HttpSession session = request.getSession(false);

        if (session == null) {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
            return;
        }

        User user = (User) session.getAttribute("user");
        if (user != null && ALLOWED_ROLES.contains(user.getRole())) {
            chain.doFilter(req, res);
        } else if (user != null) {
            response.sendError(HttpServletResponse.SC_FORBIDDEN);
        } else {
            response.sendRedirect(request.getContextPath() + "/login.jsp");
        }
    }
}
```

### 3.2 index.jsp菜单（关键片段）

```jsp
<c:if test="${sessionScope.user.role == 'MANAGER'
              || sessionScope.user.role == 'ADMIN'}">
    <a href="${pageContext.request.contextPath}/report/dashboard.jsp">查看报表</a>
</c:if>
<c:if test="${sessionScope.user.role == 'ADMIN'}">
    <a href="${pageContext.request.contextPath}/admin/panel.jsp">管理后台</a>
</c:if>
```

## 五、AI使用说明

在设计`ReportAuthFilter`时，我使用Cursor询问了"如何让Filter支持多个角色"，AI建议使用`Set<String>`存储允许的角色列表，比写多个if-else更优雅。我采纳了这个建议，因为它确实提高了代码的可扩展性——未来增加新角色只需修改Set即可。

## 六、实验总结

三级权限的关键挑战是**Filter的复用性**。如果每个路径写一个独立的Filter，代码会大量重复。通过使用`Set<String>`存储允许角色，可以让Filter更通用。未来如果角色更多，可以考虑将"路径→角色"映射存储在配置文件或数据库中，实现真正的动态RBAC。
