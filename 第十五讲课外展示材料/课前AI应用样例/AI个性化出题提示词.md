# 课前AI应用：个性化实验题目生成提示词

> 用途：在录制课外展示视频时，演示AI如何生成三级难度的个性化实验题目
> 使用工具：Claude / GPT-4

---

## 提示词（完整版，可直接复制使用）

```
你是一位大学Java Web课程的实验题目设计助手。请为第15讲"用户认证与权限控制"设计三个难度级别的个性化实验题目。

## 课程信息
- 课程名称：Java Web应用开发
- 技术栈：Java 8+、Servlet 4.0、JSP + JSTL、Maven、Tomcat
- 本讲知识点：Authentication、Authorization、Session-Based Login、RBAC、Filter、Password Hash

## 学生分层
- A层（优秀，约20%）：基础扎实，可挑战综合性和安全性任务
- B层（中等，约60%）：核心概念基本掌握，需巩固授权和Session管理
- C层（需加强，约20%）：基础较弱，需重点掌握认证流程

## 出题要求
1. 每个难度使用不同的业务场景（避免学生跨级别抄袭）
2. 每道题必须包含：任务描述、功能要求、知识点考核清单、验收标准
3. 三个级别的知识点呈递进关系：
   - 基础级：Authentication + Session-Based Login
   - 进阶级：基础级 + Authorization + RBAC + Filter
   - 挑战级：进阶级 + Password Hash + Session安全加固
4. 每个业务场景要足够具体，让学生无法直接搜到现成代码
5. 验收标准必须可测试、可验证

请为每个级别生成一道题目。
```

## 预期AI输出样例（供视频展示和录课参考）

### 基础级题目（C层学生）

**业务场景：** 校园失物招领系统

**任务描述：**
实现一个校园失物招领系统的用户登录模块。系统有两类用户：发布者（发布失物信息）和普通用户（浏览信息）。要求实现完整的登录与登出功能。

**功能要求：**
1. 登录页面（`login.jsp`）：用户名+密码输入框，登录失败显示提示信息
2. 登录处理（`LoginServlet`）：验证用户名密码，成功后将用户信息存入Session
3. 首页（`index.jsp`）：显示当前登录用户信息，未登录显示"请登录"
4. 登出处理（`LogoutServlet`）：彻底清除登录状态，重定向到登录页

**知识点考核清单：**
- [ ] 正确实现基于Session的登录状态保持
- [ ] 登录失败时给出合理反馈
- [ ] 登出时彻底销毁Session
- [ ] 页面根据登录状态显示不同内容

**验收标准：**
1. 输入正确账号密码 → 登录成功，跳转首页，显示用户信息
2. 输入错误密码 → 登录失败，显示错误提示
3. 点击退出 → 回到登录页，再次访问首页显示"请登录"
4. 退出后直接访问首页 → 不显示任何用户信息

### 进阶级题目（B层学生）

**业务场景：** 实验室设备借用管理系统

**任务描述：**
实现一个实验室设备借用管理系统的用户认证与权限控制模块。系统有三类用户：管理员（管理设备信息）、教师（审批借用申请）、学生（提交借用申请）。要求实现登录、登出，以及基于角色的权限控制。

**功能要求：**
1. 登录页面 + 登录处理（同基础级）
2. 首页根据角色显示不同菜单：
   - 管理员：显示"设备管理"、"用户管理"
   - 教师：显示"审批借用"
   - 学生：显示"我的借用"
3. 权限过滤器（`RoleBasedAuthFilter`）：
   - `/admin/*` 路径仅管理员可访问
   - `/teacher/*` 路径仅教师和管理员可访问
   - 未授权访问返回403或重定向
4. 登出时彻底清除所有状态

**知识点考核清单：**
- [ ] 基础级全部知识点
- [ ] 正确实现基于角色的菜单显示（前端）
- [ ] 正确实现基于角色的路径保护（后端Filter）
- [ ] 区分Authentication和Authorization
- [ ] 多级角色权限的层级关系

**验收标准：**
1. 基础级全部验收标准
2. 学生登录 → 只看到"我的借用"，访问/admin/* → 403
3. 教师登录 → 看到"审批借用"，访问/admin/* → 403
4. 管理员登录 → 看到全部菜单，访问/admin/* → 正常

### 挑战级题目（A层学生）

**业务场景：** 在线考试成绩管理系统

**任务描述：**
实现一个在线考试成绩管理系统的完整认证与授权模块。在进阶级要求的基础上，增加密码安全存储和Session安全加固。

**功能要求：**
1. 进阶级全部功能
2. 密码使用 BCrypt 加密存储（不使用明文或简单SHA-256）
3. 登录成功后重新生成Session ID（防止固定会话攻击）
4. Session设置合理超时时间（30分钟）
5. 关键操作记录审计日志（登录成功、登录失败、访问拒绝）

**知识点考核清单：**
- [ ] 进阶级全部知识点
- [ ] BCrypt密码哈希与验证
- [ ] Session固定攻击防护
- [ ] Session超时配置
- [ ] 安全审计日志

**验收标准：**
1. 进阶级全部验收标准
2. 数据库/内存中存储的密码为BCrypt哈希值
3. 登录前后Session ID不同（可通过浏览器开发者工具验证）
4. 30分钟无操作后Session过期，需重新登录
5. 控制台/日志文件中可看到登录和访问记录
