# 第三讲教学设计：JSP核心技术—编写动态页面 (20分钟)

## 课程信息
- **课程名称**: Java Web应用开发 (Java Web Application Development)
- **教学方式**: 双语教学（英文授课，关键术语中英文对照）
- **教学主题**: JSP核心技术：编写动态页面
- **授课时长**: 20分钟
- **适用对象**: 已掌握Servlet基础的Java Web开发初学者

---

## 一、教学目标

### 知识目标
1.  理解JSP（JavaServer Pages）作为视图层（View Layer）技术的根本目的：分离动态逻辑与静态显示。
2.  掌握JSP的三大核心语法：表达式Expression（`<%= ... %>`）、脚本片段Scriptlet（`<% ... %>`）和指令Directive（`<%@ ... %>`）。
3.  了解JSP的生命周期（Lifecycle），特别是它被Tomcat容器翻译成Servlet的核心原理。
4.  初步认识JSP九大隐含对象（Implicit Objects），重点理解`request`、`response`和`out`的直接可用性。

### 能力目标
1.  能够创建一个JSP页面，并嵌入简单的Java表达式来显示动态数据。
2.  能够在JSP页面中使用脚本片段（Scriptlet）实现简单的逻辑控制（如循环、判断）。
3.  能够使用`<%@ page ... %>`指令设置页面属性，如`contentType`和`import`。

### 素质目标
1.  建立"关注点分离"（Separation of Concerns）的软件工程思想。
2.  培养编写可读性、可维护性更强的视图代码的意识。
3.  **[思政融合]** 介绍JSP技术时，可引申讨论自主知识产权技术的重要性，以及掌握核心技术对于国家信息安全和科技自立的意义。

## 二、学情分析

### 知识基础
- **已掌握**：HTTP协议的请求/响应模型、Servlet的编写与部署、`@WebServlet`注解映射、`HttpServletRequest`和`HttpServletResponse`对象的基本使用、Servlet生命周期
- **待建立**：JSP页面的语法体系（表达式、脚本片段、指令）、JSP与Servlet的内在关系（JSP本质是Servlet）、"关注点分离"的软件设计思想

### 能力水平
- 学生已能编写简单的Servlet处理请求并返回动态内容，但体验过在`out.println()`中拼接HTML的繁琐
- 学生理解请求-响应模型，能够将HTML表单与Servlet正确关联，具备前后端数据交互的初步能力
- 学生在上一讲中已使用Cursor辅助编写Servlet代码，对AI代码补全有初步体验，能够基本判断AI建议的合理性

### AI素养现状
- 经过前两讲的递进训练，学生已掌握两种AI应用模式：L1的"ChatGPT生成文本示例并对比验证"和L2的"Cursor辅助编码并选择性采纳建议"。具体能力基线：约65%的学生能在Cursor中自主判断AI补全建议的合理性（如识别多余的import语句）；约55%的学生已能在提示词中包含格式要求和约束条件；约40%的学生在L2中至少主动拒绝过1条不合理的AI建议。学生的AI使用模式已从L1的"被动接受"初步过渡到L2的"选择性采纳"
- 本讲将进一步提升AI应用层次（L3）：从"AI辅助编码"升级为"**AI生成模板 + 学生审查安全隐患**"，这是AI使用能力的质变节点。学生此前使用AI的场景均为"让AI帮我做对"（正向辅助），本讲**首次要求学生"找出AI做错的地方"**（逆向审查），是批判性思维能力的关键跃迁。同时，本讲是学生首次在Cursor中使用**内联对话功能生成完整代码模板**（而非仅接受逐行补全），需要教师演示"Ctrl+K生成代码块"的操作方式
- 学生对Web安全（如XSS攻击）尚无认知，不具备从安全角度审查代码的能力。前期调研显示：超过90%的学生从未听说过XSS或SQL注入等Web安全概念；在L2的Servlet编码中，没有学生主动考虑过输入验证或输出转义。本讲需要教师通过具体攻击演示（如在输入框中注入`<script>alert('hacked')</script>`标签并观察页面弹窗效果）建立安全意识后，再引导学生将安全审查应用到AI生成的JSP模板中

## 三、教学重点与难点

### 教学重点
-   JSP与Servlet在职责上的核心区别：JSP主外（负责展示），Servlet主内（负责控制）。
-   JSP表达式（`<%= ... %>`）的简洁用法。
-   `<%@ page ... %>`指令，特别是`contentType="text/html;charset=UTF-8"`的必要性。
-   JSP的生命周期原理（JSP -> .java -> .class）。

### 教学难点
-   **概念难点**: 真正理解JSP文件本身不是直接运行的，而是作为模板被服务器动态翻译成一个Servlet类来运行。
-   **实践难点**: 在JSP中混合使用HTML标签和JSP脚本时，容易产生语法错误，需要理解代码的执行时机。

## 四、教学内容与时间安排

### **第一部分：问题引入 - 解救"意大利面条式"代码 (3分钟)**

#### 内容要点
1.  **回顾痛点 (Recalling the Pain Point):**
    *   展示上一节课`LoginServlet`的代码，特别是`out.println()`中拼接大量HTML的部分。
    *   **提问**: "同学们，请看这段代码。如果我们想给欢迎页面加一个复杂的表格、一些CSS样式，甚至一些JavaScript，会发生什么？我们的Java类会变成一锅混杂着Java逻辑、HTML结构、CSS样式的'意大利面条'！"
2.  **场景冲突 (Scenario Conflict):**
    *   **趣味故事**: "想象一下，一位UI设计师（小美）设计好了一个精美的HTML页面，她交给Java工程师（小明）。小明需要把这个HTML一行行地复制粘贴到他的`out.println()`语句里。第二天，小美说要改个颜色，小明又要在一大堆Java代码里找到那几行HTML去修改。这种合作方式，是不是一场灾难？"
3.  **引出解决方案 (Introducing the Solution):**
    *   **思路转变**: 我们能不能反过来？在一个HTML文件里，只在需要动态内容的地方"挖个坑"，然后用Java代码把数据"填进去"？
    *   这种"以HTML为中心，Java为点缀"的技术，就是 **JSP (JavaServer Pages)**。

#### 教学方法
-   故事化教学，增强代入感。
-   对比法，强化JSP解决Servlet痛点的优势。

### **第二部分：第一个JSP页面 - 从静态到动态 (8分钟)**

#### 内容要点
1.  **修改Servlet (Modifying the Servlet):**
    *   改造上节课的`LoginServlet`，让它不再直接输出HTML，而是将数据准备好，然后"请求转发"给一个JSP页面去显示。
        ```java
        // In LoginServlet's doGet method
        String agentName = request.getParameter("agentName");
        // 将特工代号存入request这个"包裹"里，准备传递
        request.setAttribute("agent", agentName); 
        // 请求转发给mission.jsp去显示
        request.getRequestDispatcher("/mission.jsp").forward(request, response);
        ```
2.  **创建第一个JSP (Creating the First JSP):**
    *   在`webapp`目录下创建一个`mission.jsp`文件。
    *   **第一步：先写纯HTML结构**
        ```jsp
        <%@ page contentType="text/html;charset=UTF-8" language="java" %>
        <html>
        <head><title>任务简报</title></head>
        <body>
            <h1>特工任务简报 (Mission Briefing)</h1>
            <hr>
            <h3>欢迎您，[这里应该是动态的特工代号]</h3>
            <p>您的任务书正在生成中...</p>
            <p>当前服务器时间是：[这里应该是动态的服务器时间]</p>
        </body>
        </html>
        ```
3.  **填坑 - 使用JSP表达式 (Filling the Blanks with JSP Expressions):**
    *   **获取数据**: 使用脚本片段（Scriptlet）从`request`中取出数据。
        `<% String name = (String) request.getAttribute("agent"); %>`
    *   **显示数据**: 使用JSP表达式`<%= ... %>`来显示变量。
        `<h3>欢迎您，<%= name %></h3>`
    *   **显示动态时间**: `当前服务器时间是：<%= new java.util.Date() %>`
4.  **JSP生命周期揭秘 (Demystifying the Lifecycle):**
    *   **核心原理**: 第一次访问`mission.jsp`时，Tomcat会在其内部工作目录里，把`mission.jsp`文件**翻译**成一个`mission_jsp.java`的Servlet文件，然后再编译成`.class`文件来运行。
    *   展示一张清晰的流程图: **`mission.jsp` --> `mission_jsp.java` --> `mission_jsp.class`**。
    *   **结论**: JSP本质上就是一种编写Servlet的便捷方式！它让我们专注于页面长什么样。

#### 教学方法
-   **代码重构**: 引导学生亲历从Servlet到JSP的改造过程。
-   **原理图示**: 用图清晰展示JSP的底层工作原理，破除神秘感。

### **第三部分：JSP核心语法 - 动态能力的来源 (6分钟)**

#### 内容要点
1.  **脚本片段（Scriptlets `<% ... %>`）:**
    *   用于编写任意Java代码块。
    *   **趣味案例**: "假设总部发来了多条任务指令，我们需要用一个列表展示出来。"
        ```jsp
        <%-- 在JSP中创建一个任务列表 --%>
        <%
            java.util.List<String> missions = new java.util.ArrayList<>();
            missions.add("任务一：学习JSP表达式");
            missions.add("任务二：掌握JSP脚本");
            missions.add("任务三：理解JSP生命周期");
        %>
        <h3>您的任务列表：</h3>
        <ul>
        <% for(String mission : missions) { %>
            <li><%= mission %></li>
        <% } %>
        </ul>
        ```
    *   **强调**: 这种写法虽然功能强大，但已经开始让HTML和Java代码再次混合，为下一课埋下伏笔。
2.  **指令（Directives `<%@ ... %>`）:**
    *   是给JSP翻译引擎看的"指令"，用来设置整个页面的属性。
    *   **`<%@ page ... %>`**: 重点讲解`contentType="text/html;charset=UTF-8"`（防止中文乱码）和`import="java.util.List, java.util.ArrayList"`（导入Java类）。
3.  **隐含对象（Implicit Objects）:**
    *   讲解JSP的一大便利之处：有9个可以直接使用的、预先定义好的对象，无需我们自己创建。
    *   重点介绍：`request`, `response`, `session`, `application`, `out`。强调`request`和`response`就是从Servlet传递过来的那两个对象。

#### 教学方法
-   **功能驱动**: 以"显示任务列表"的功能需求来自然地引出循环脚本。
-   **分类讲解**: 将JSP语法清晰地分为表达式、脚本、指令三类。

### **第四部分：总结与新的挑战 (3分钟)**

#### 总结要点
1.  **JSP是"皮"**: 以HTML为中心，是视图模板。
2.  **Servlet是"芯"**: 以Java为中心，负责逻辑控制。
3.  **JSP的本质**: 它是一种能被自动翻译成Servlet的特殊文本文件。
4.  **JSP的语法**: 表达式`<%= ... %>`负责输出，脚本`<% ... %>`负责逻辑。

#### 提出新的挑战 (The New Challenge)
-   再次展示包含`for`循环的JSP代码。
-   **提问**: "同学们，虽然我们把显示逻辑从Servlet移到了JSP，看起来好多了。但请看这里，我们的HTML页面里是不是又出现了`<% for(...) { ... } %>`这样的纯Java代码？如果设计师小美想调整这个列表的样式，她看到这些符号是不是又会头疼？"
-   **展望**: "有没有一种更优雅、更像'标签'、而不是像'代码'的方式来完成这些循环和判断呢？当然有！下一节课，我们将学习EL表达式和JSTL标签库，它们将彻底把Java代码从我们的视图中驱逐出去，实现真正的'前后端分离'！"

## 六、AI应用环节说明

### 6.1 使用工具

| 工具 | 使用者 | 场景 |
|:---|:---|:---|
| Cursor（AI代码编辑器） | 教师 | 实验设计阶段：使用Cursor生成JSP页面模板，作为学生安全审查的实验素材 |
| Cursor（AI代码编辑器） | 学生 | 实验实践阶段：审查AI生成的JSP模板，识别其中的安全隐患（如未转义输出） |

### 6.2 应用环节
AI工具主要应用于**第二部分"第一个JSP页面—从静态到动态"**（约第4~11分钟）及**第三部分"JSP核心语法"**（约第12~17分钟）的实验设计阶段。教师使用Cursor生成一个包含动态数据展示的JSP页面模板（如显示用户提交的姓名和留言），然后将该模板分发给学生。学生的任务不是直接使用该模板，而是审查其中是否存在安全隐患——例如AI生成的模板中可能直接使用`<%= request.getParameter("name") %>`输出用户输入而未进行HTML转义，存在跨站脚本攻击（XSS）风险。

### 6.3 设计目的
1. **将AI输出作为"教学反面教材"**：AI生成的JSP模板通常语法正确但缺乏安全意识，这恰好为学生提供了一个真实的安全审查场景，比教师人为构造的错误更具说服力
2. **培养安全编码意识**：JSP直接嵌入动态内容的特性使其天然面临XSS等安全风险，通过审查AI模板让学生在学习语法的同时建立安全编码思维
3. **深化对JSP语法的理解**：学生在审查AI生成的代码时，必须逐行理解表达式、脚本片段、指令的作用，这比被动阅读教材更能促进深层理解

### 6.4 操作流程
```
第1步：教师使用Cursor生成一个"特工任务留言板"JSP页面模板，
      包含表达式输出、脚本片段循环、page指令等核心语法元素
     ↓
第2步：教师将AI生成的JSP模板展示给学生（通过课件或共享编辑器），
      要求学生逐行阅读并标注每个JSP语法元素的类型和作用
     ↓
第3步：教师提出挑战问题："如果用户在留言框中输入<script>alert('hacked')</script>，
      这个页面会发生什么？"引导学生发现AI模板中的XSS漏洞
     ↓
第4步：学生分组讨论修复方案，教师引导学生了解输出转义的必要性，
      并对比AI原始模板与安全修复后的版本，巩固JSP语法理解
```

**第1步教师生成模板的提示词（Cursor内联对话）**:
> "使用JSP语法创建一个'特工任务留言板'页面：包含<%@ page %>指令设置UTF-8编码，使用Scriptlet从request中获取用户提交的姓名和留言内容，用JSP表达式<%= %>直接显示在页面上，并用Scriptlet循环展示一个任务列表。不要添加任何安全处理或输入过滤。"

**第3步学生安全审查引导提示词**:
> "请审查以下JSP代码的安全性，特别关注用户输入的处理方式。如果存在XSS（跨站脚本攻击）风险，请指出具体代码行并给出修复方案：[粘贴AI生成的JSP模板]"

### 6.5 预期效果

| 维度 | 传统课堂 | AI辅助课堂 |
|:---|:---|:---|
| 语法学习方式 | 学生被动跟随教师逐条讲解JSP语法规则 | 学生通过审查AI生成的真实JSP代码主动识别和理解各语法元素 |
| 安全意识培养 | 安全编码通常在后续高级课程中才涉及，初学阶段容易忽略 | 通过审查AI模板中的安全漏洞，在学习JSP语法的同时即建立安全编码意识 |
| 批判性思维 | 学生倾向于认为"能运行的代码就是正确的代码" | 学生认识到语法正确不等于安全正确，建立对代码质量的多维评价标准 |

**可量化指标**：
| 指标 | 传统课堂预期 | AI辅助课堂预期 | 测量方式 |
|:---|:---|:---|:---|
| 学生能正确识别JSP三大语法元素的比例 | 50%-60% | 75%-85% | 课堂语法标注练习 |
| 学生能发现AI模板中XSS漏洞的比例 | — | 60%-70% | 安全审查实验报告 |
| 课后作业中主动进行输出转义的学生比例 | 10%-15% | 40%-50% | 课后JSP作业代码审查 |

### 6.6 防滥用措施
1. **任务限定**：AI仅用于生成JSP页面模板作为审查素材，不允许学生直接使用AI生成的模板提交作业，必须在理解和修复安全问题后自行编写
2. **批判性评估**：要求学生在审查报告中明确标注AI生成模板中的至少一个安全隐患，并给出修复建议和修复后的代码
3. **教师审核**：教师预先审查AI生成的模板，确保其中包含适当难度的安全问题（如未转义输出、硬编码敏感信息等），同时确保模板的语法覆盖本讲教学重点
4. **AI使用边界**：AI是辅助者而非替代者，学生必须理解JSP表达式、脚本片段、指令的语法和执行原理后再参考AI输出，最终评价以学生能否独立编写安全的JSP页面为准

### AI辅助评价
- **AI辅助出题与测验**：教师使用Cursor快速生成不同复杂度的JSP代码片段（含表达式、脚本片段、指令的混合使用），要求学生判断代码输出结果或指出潜在的语法/安全问题
- **自动化安全扫描**：学生提交的JSP页面作业通过AI工具进行自动化安全扫描，检测未转义的`<%= %>`表达式输出、硬编码敏感信息等问题，生成安全审查报告辅助教师评分
- **L3 安全审查能力分级评估**（首次逆向审查AI输出基准）：
  - **L3-优秀**：能独立定位AI生成模板中的全部XSS风险点，给出合理修复方案（采用HTML实体转义而非简单删除特殊字符），并能解释漏洞原理（占比预期10%-15%）
  - **L3-达标**：能在教师引导下发现AI模板中至少1处安全隐患，修复方案基本合理；能正确区分JSP三大语法元素的类型和作用（占比预期50%-60%）
  - **L3-待提升**：未能发现AI模板中的安全问题，或将语法正确的代码误判为有安全风险；对JSP语法元素的分类仍存在混淆（占比预期25%-35%）
