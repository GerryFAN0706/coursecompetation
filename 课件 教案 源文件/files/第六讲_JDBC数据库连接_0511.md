## JDBC编程六步曲

###### Six Steps of JDBC Programming

**必须掌握的六个步骤 (Six Essential Steps)**:

<!-- _class: cols2_ol_ci -->

- **步骤1** 加载驱动  
  *Load driver*
- **步骤2** 建立连接  
  *Establish connection*
- **步骤3** 创建Statement  
  *Create statement*
- **步骤4** 执行SQL  
  *Execute SQL*
- **步骤5** 处理结果  
  *Process results*
- **步骤6** 关闭资源  
  *Close resources*

**记忆口诀**: 加载→连接→创建→执行→处理→关闭  
*Mnemonic: Load→Connect→Create→Execute→Process→Close*

## 步骤1：加载驱动

###### Step 1: Load Driver

**代码 (Code)**:
```java
Class.forName("com.mysql.cj.jdbc.Driver");
```

**作用 (Purpose)**:
- 请求JVM加载MySQL驱动类 *Load driver class*
- 驱动注册到DriverManager *Register to DriverManager*

**说明 (Note)**:
- JDBC 4.0+ 可自动加载（SPI机制）
- 显式加载有助于理解原理

**其他数据库驱动 (Other Drivers)**:
- Oracle: `oracle.jdbc.driver.OracleDriver`
- SQL Server: `com.microsoft.sqlserver.jdbc.SQLServerDriver`

## 步骤2：建立连接

###### Step 2: Establish Connection

**代码 (Code)**:
```java
Connection conn = DriverManager.getConnection(url, user, password);
```

**连接URL格式 (URL Format)**:
```java
String url = "jdbc:mysql://localhost:3306/mydatabase" +
             "?useSSL=false&serverTimezone=UTC";
String user = "root";
String password = "123456";
```

**URL解析 (URL Breakdown)**:
- `jdbc:mysql://` - 协议 *Protocol*
- `localhost:3306` - 主机:端口 *Host:Port*
- `mydatabase` - 数据库名 *Database name*
- `?参数` - 连接参数 *Parameters*

## 步骤3：创建Statement

###### Step 3: Create Statement

**两种Statement (Two Types)**:

<!-- _class: cols-2 -->

<div class="ldiv">

**Statement**
```java
Statement stmt = 
  conn.createStatement();
```

**特点**:
- ❌ 字符串拼接SQL
- ❌ 有SQL注入风险
- ⚠️ 不推荐使用

</div>

<div class="rdiv">

**PreparedStatement** ⭐推荐
```java
String sql = 
  "SELECT * FROM users WHERE id=?";
PreparedStatement pstmt = 
  conn.prepareStatement(sql);
pstmt.setInt(1, userId);
```

**特点**:
- ✅ 预编译SQL
- ✅ 防止SQL注入
- ✅ 性能更好

</div>

## 步骤4：执行SQL

###### Step 4: Execute SQL

**两种执行方法 (Two Execution Methods)**:

**查询 (Query)** - 使用 `executeQuery()`
```java
String sql = "SELECT * FROM users WHERE id = ?";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setInt(1, userId);
ResultSet rs = pstmt.executeQuery();  // 返回结果集
```

**更新 (Update)** - 使用 `executeUpdate()`
```java
String sql = "INSERT INTO users (name, age) VALUES (?, ?)";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setString(1, "张三");
pstmt.setInt(2, 25);
int rows = pstmt.executeUpdate();  // 返回影响行数
```

## 步骤5：处理结果集

###### Step 5: Process ResultSet

**ResultSet游标机制 (ResultSet Cursor)**:

```java
ResultSet rs = pstmt.executeQuery();

while (rs.next()) {  // 移动游标到下一行
    // 按列名获取数据 (Get data by column name)
    int id = rs.getInt("id");
    String name = rs.getString("name");
    int age = rs.getInt("age");
    
    System.out.println(id + " - " + name + " - " + age);
}
```

**获取数据方法 (Get Methods)**:
- `getInt()` - 获取整数 *Get integer*
- `getString()` - 获取字符串 *Get string*
- `getDouble()` - 获取浮点数 *Get double*
- `getDate()` - 获取日期 *Get date*

## 步骤6：关闭资源

###### Step 6: Close Resources

**资源关闭原则 (Resource Closing Principle)**:

**原则 (Principles)**:
- 🔴 **后开先关** *Last-in, first-out*
- 🔴 **必须关闭** *Must close*
- 🔴 **在finally中** *In finally block*

**正确写法 (Correct Way)**:
```java
Connection conn = null;
PreparedStatement pstmt = null;
ResultSet rs = null;

try {
    // 1-5步骤...
} catch (SQLException e) {
    e.printStackTrace();
} finally {
    // 关闭资源 (Close resources)
    if (rs != null) try { rs.close(); } catch (SQLException e) {}
    if (pstmt != null) try { pstmt.close(); } catch (SQLException e) {}
    if (conn != null) try { conn.close(); } catch (SQLException e) {}
}
```

## 完整JDBC查询示例

###### Complete JDBC Query Example

```java
public List<User> queryUsers() {
    Connection conn = null;
    PreparedStatement pstmt = null;
    ResultSet rs = null;
    List<User> users = new ArrayList<>();
    
    try {
        // 1. 加载驱动 (已自动加载)
        // 2. 建立连接 (Establish connection)
        conn = DriverManager.getConnection(url, user, pwd);
        
        // 3. 创建Statement (Create statement)
        String sql = "SELECT * FROM users WHERE age > ?";
        pstmt = conn.prepareStatement(sql);
        pstmt.setInt(1, 18);
        
        // 4. 执行SQL (Execute SQL)
        rs = pstmt.executeQuery();
        
        // 5. 处理结果 (Process results)
        while (rs.next()) {
            User u = new User();
            u.setId(rs.getInt("id"));
            u.setName(rs.getString("name"));
            users.add(u);
        }
    } catch (SQLException e) {
        e.printStackTrace();
    } finally {
        // 6. 关闭资源 (Close resources)
        // ... close rs, pstmt, conn ...
    }
    return users;
}
```

## PreparedStatement vs Statement

###### PreparedStatement vs Statement

**为什么用PreparedStatement？** *Why PreparedStatement?*

<!-- _class: cols-2 -->

<div class="ldiv">

**Statement - 危险！**
```java
String sql = "SELECT * FROM users " +
  "WHERE name='" + userInput + "'";
stmt.executeQuery(sql);
```

**SQL注入攻击 (SQL Injection)**:
```
输入: ' or '1'='1
结果SQL: 
SELECT * FROM users 
WHERE name='' or '1'='1'
```
❌ 返回所有用户！

</div>

<div class="rdiv">

**PreparedStatement - 安全！**
```java
String sql = "SELECT * FROM users " +
  "WHERE name=?";
pstmt = conn.prepareStatement(sql);
pstmt.setString(1, userInput);
pstmt.executeQuery();
```

**优势 (Advantages)**:
- ✅ 防止SQL注入
- ✅ 预编译提升性能
- ✅ 代码更清晰
- ✅ 类型安全

</div>

## 封装JDBC工具类

###### Encapsulate JDBC Utility

**问题**: 重复代码太多 *Problem: Too much repetition*

**解决方案**: 创建 `DBUtil` 工具类

```java
public class DBUtil {
    private static final String URL = "jdbc:mysql://localhost:3306/mydb";
    private static final String USER = "root";
    private static final String PWD = "123456";
    
    // 获取连接 (Get connection)
    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PWD);
    }
    
    // 关闭资源 (Close resources)
    public static void close(Connection conn, Statement stmt, ResultSet rs) {
        if (rs != null) try { rs.close(); } catch (SQLException e) {}
        if (stmt != null) try { stmt.close(); } catch (SQLException e) {}
        if (conn != null) try { conn.close(); } catch (SQLException e) {}
    }
}
```
