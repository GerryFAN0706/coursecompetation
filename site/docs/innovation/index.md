# AI教学创新设计

## 真实问题（Problem-Driven）

课程改革针对四类真实痛点：

1. 实验任务同质化严重，难以有效识别真实学习水平。
2. 教师逐份代码审查成本高，反馈滞后。
3. 学生“能跑起来”不等于真正掌握关键知识点。
4. 统一难度无法兼顾分层学情。

## 改革目标

| 改革目标 | 实现路径 |
|---|---|
| 消减抄袭与高相似度提交 | AI一人一题，保持同知识点异场景 |
| 提升反馈速度与质量 | AI初审 + 教师纠偏 + 测试验证 |
| 从结果评价转向过程评价 | 知识点对照 + 代码质量 + 安全 + 规范 |
| 支持分层学习与持续改进 | 基础/进阶/挑战任务与个性化建议 |

## AI赋能总架构

```text
课前
  学情分析 -> 分层任务生成 -> 实验手册与测试清单
课中
  AI代码审查 -> 小组讨论 -> 教师纠偏 -> 修改与验证
课后
  智能反馈报告 -> 二次提交 -> 数据驱动精准干预
```

## 人机协同定位

- AI 不是替代教师和学生，而是“诊断、对照、反馈”的智能支撑系统。
- 教师职责从“批改者”升级为“设计者、引导者、干预者”。
- 学生职责从“被动接收”转向“主动判断、修改、验证、反思”。
