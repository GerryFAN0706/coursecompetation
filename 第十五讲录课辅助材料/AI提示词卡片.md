# AI 提示词卡片

> 使用说明：AI 助教的任务不是帮你写答案，而是帮你更快地发现问题、对照知识点并生成验证方案。
> AI 的输出是"候选意见"，不是"标准答案"——是否采纳，由你判断。

---

## 1. 教师总控提示词（教师投影演示用）

```
请审查这个 Java Web 用户认证与权限控制模块，重点检查以下方面：

1. 认证流程是否正确
2. Session 的创建、保存、注销是否规范
3. 是否存在未授权访问或越权访问风险
4. RoleBasedAuthFilter 是否真正实现了基于角色的访问控制
5. JSP 菜单渲染是否与服务端权限控制一致
6. 密码处理是否存在安全隐患
7. 是否可能存在固定会话攻击、会话残留或重复提交问题

请按以下格式输出：
- 问题位置
- 风险说明
- 对应课程知识点
- 修改建议
- 建议验证方式
```

---

## 2. 知识点对照提示词（各组通用）

```
请根据以下 Java Web 课堂知识点，判断这段代码中哪些知识点已经正确体现，哪些存在问题：

知识点清单：
1. Authentication（认证）
2. Authorization（授权）
3. Session-Based Login（基于会话的登录）
4. RBAC（基于角色的访问控制）
5. Filter（过滤器）
6. Password Hash（密码哈希）

请按以下格式输出：
- 知识点名称
- 当前情况（已正确体现 / 部分体现 / 存在问题）
- 代码依据
- 改进建议
```

---

## 3. 修改建议提示词（修改阶段用）

```
我已经知道这个 Java Web 权限控制模块存在问题。请不要直接重写整段代码，而是只告诉我：

1. 最优先应修改的 3 个点
2. 每个修改点的理由
3. 每个修改点对应的课程知识点
4. 修改后必须验证的系统行为
```

---

## 4. 测试清单生成提示词（验证阶段用）

```
请为这个 Java Web 登录与权限控制模块生成一个最小验证清单，要求覆盖以下场景：

1. 未登录访问后台
2. 普通用户访问后台
3. 管理员访问后台
4. 注销后再次访问后台
5. JSP 菜单显示与角色一致性

请按"测试步骤 + 预期结果"格式输出，尽量简洁。
```

---

## 5. 教师纠偏提示词（AI 输出过长时用）

```
请把这个 Java Web 认证与授权问题的分析结果压缩为课堂上最值得强调的 3 个问题，每个问题只保留：

1. 问题本身
2. 为什么危险
3. 课堂上最应该提醒学生的知识点
```

---

## 6. AI 输出追问提示词

**当 AI 太笼统时：**
```
请不要泛泛而谈，只指出当前代码中最关键的 2 个安全问题，并分别说明：
1. 为什么危险
2. 该如何验证问题是否存在
```

**当 AI 直接给整段代码时：**
```
请不要直接重写整段代码，只指出需要修改的具体位置、修改理由和最小修改方案。
```

**当 AI 未对照知识点时：**
```
请把你的分析结果与以下知识点逐项对应：
Authentication、Authorization、Session-Based Login、RBAC、Filter、Password Hash
```
