## 认证与授权

###### Authentication & Authorization

**两个不同的概念 (Two Different Concepts)**:

<!-- _class: cols-2 -->

<div class="ldiv">

**认证 (Authentication)**

**问题**: 你是谁？  
*Who are you?*

**过程**:
- 用户提供凭证（用户名+密码）
- 系统验证身份
- 确认用户身份

**比喻**: 身份证检查  
*ID verification*

</div>

<div class="rdiv">

**授权 (Authorization)**

**问题**: 你能做什么？  
*What can you do?*

**过程**:
- 检查用户角色/权限
- 判断是否有访问权限
- 允许或拒绝访问

**比喻**: 门禁卡权限  
*Access card permission*

</div>

**顺序**: 先认证，后授权  
*Order: Authenticate first, then authorize*

## 密码安全

###### Password Security

**❌ 不要明文存储密码！** *Never store plaintext passwords!*

**密码散列 (Password Hashing)**:

```java
// 注册时：存储散列值
String hash = PasswordUtil.sha256(password);
user.setPasswordHash(hash);
dao.insert(user);

// 登录时：比对散列值
User user = dao.findByUsername(username);
if (PasswordUtil.matches(password, user.getPasswordHash())) {
    // 登录成功
}
```

**推荐算法**: BCrypt, Argon2 (带盐值)

## 登录认证流程

###### Login Authentication Flow

**LoginServlet实现**:

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    private UserService service = new UserService();
    
    protected void doPost(...) {
        String username = request.getParameter("username");
        String password = request.getParameter("password");
        
        User user = service.login(username, password);
        
        if (user != null) {
            // 登录成功，保存到Session
            request.getSession().setAttribute("user", user);
            response.sendRedirect("index.jsp");
        } else {
            // 登录失败
            request.setAttribute("error", "用户名或密码错误");
            request.getRequestDispatcher("login.jsp").forward(...);
        }
    }
}
```

## 登出功能

###### Logout Function

**LogoutServlet实现**:

```java
@WebServlet("/logout")
public class LogoutServlet extends HttpServlet {
    protected void doGet(...) {
        HttpSession session = request.getSession(false);
        if (session != null) {
            session.invalidate();  // 销毁Session
        }
        response.sendRedirect("login.jsp");
    }
}
```

**效果 (Effect)**:
- 清除Session中的用户信息
- 用户需要重新登录

## RBAC模型

###### RBAC Model

**基于角色的访问控制** *Role-Based Access Control*

**角色定义 (Role Definition)**:
- 👤 **USER** - 普通用户 *Regular user*
- 👨‍💼 **MANAGER** - 管理员 *Manager*
- 👑 **ADMIN** - 超级管理员 *Administrator*

**权限分配 (Permission Assignment)**:
- USER: 可访问 `/user/*`
- MANAGER: 可访问 `/user/*`, `/report/*`
- ADMIN: 可访问所有路径

**原则**: 最小权限、默认拒绝  
*Principle: Least privilege, default deny*

## 权限过滤器

###### Authorization Filter

**基于角色的Filter** *Role-based filter*

```java
@WebFilter("/admin/*")
public class AdminFilter implements Filter {
    public void doFilter(...) {
        HttpSession session = request.getSession(false);
        User user = (User) session.getAttribute("user");
        
        // 检查是否登录且角色为ADMIN
        if (user != null && "ADMIN".equals(user.getRole())) {
            chain.doFilter(req, res);  // 放行
        } else {
            response.sendError(403);  // 拒绝访问
        }
    }
}
```

**拦截路径**: `/admin/*` 只允许ADMIN访问

## JSP中按角色显示

###### Role-Based Display in JSP

**动态菜单 (Dynamic Menu)**:

```jsp
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

<c:if test="${sessionScope.user != null}">
    <p>欢迎，${user.username}</p>
    <a href="logout">退出</a>
</c:if>

<c:if test="${user.role == 'ADMIN'}">
    <a href="admin/panel">管理后台</a>
</c:if>

<c:if test="${user.role == 'MANAGER'}">
    <a href="report/view">查看报表</a>
</c:if>
```

**效果**: 不同角色看到不同菜单

## 安全最佳实践

###### Security Best Practices

<!-- _class: cols2_ol_ci -->

**认证安全 (Authentication Security)**:

- **密码加密** 使用BCrypt  
  *Encrypt passwords*
- **防暴力破解** 登录限制  
  *Prevent brute force*
- **Session超时** 30分钟  
  *Session timeout*
- **HTTPS** 加密传输  
  *Encrypted transfer*

**授权安全 (Authorization Security)**:

- **最小权限** 按需分配  
  *Least privilege*
- **默认拒绝** 白名单机制  
  *Default deny*
- **越权检查** 防止横向越权  
  *Prevent privilege escalation*
- **日志审计** 记录访问  
  *Audit logging*

## 权限设计原则

###### Authorization Design Principles

**安全原则 (Security Principles)**:

<!-- _class: cols2_ol_ci -->

- **最小权限** 只给必需权限  
  *Least privilege*
- **默认拒绝** 未授权即拒绝  
  *Default deny*
- **白名单** 明确允许列表  
  *Whitelist*
- **审计日志** 记录访问  
  *Audit log*
- **角色分离** 职责明确  
  *Role separation*
- **定期审查** 权限复查  
  *Regular review*
