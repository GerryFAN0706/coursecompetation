## 阶段一：示例工程（录课前必须完成）

### 1.1 完善"问题版"示例工程
- [ ] 确认 Maven 项目可正常编译运行（Tomcat + MySQL）
- [ ] 实现最小功能集：
  - [ ] `login.jsp` 登录页面
  - [ ] `index.jsp` 首页（含导航菜单）
  - [ ] `admin/panel.jsp` 管理后台页面
  - [ ] `LoginServlet.java` 登录处理
  - [ ] `LogoutServlet.java` 登出处理
  - [ ] `RoleBasedAuthFilter.java` 权限过滤器
  - [ ] `UserService.java` + `UserDao.java` + `User.java`
  - [ ] `PasswordUtil.java`（简单哈希即可）
  - [ ] `web.xml` 配置
- [ ] 预置两个测试账户：`admin/ADMIN`、`tom/USER`
- [ ] 植入3个缺陷：
  - [ ] 缺陷1：`RoleBasedAuthFilter` 只判断 `user != null`，不判断 role
  - [ ] 缺陷2：`index.jsp` 菜单只按 `user != null` 显示管理后台入口
  - [ ] 缺陷3：`LogoutServlet` 只 `removeAttribute`，不 `invalidate()`
- [ ] 逐一验证3个错误行为可稳定复现：
  - [ ] tom/USER 登录后能直接访问 `/admin/panel.jsp`
  - [ ] tom/USER 登录后首页显示"管理后台"入口
  - [ ] 退出后刷新/重访受保护资源时行为不正常

### 1.2 准备"修复版"示例工程
- [ ] 复制问题版为修复版
- [ ] 修复3个缺陷：
  - [ ] Filter 增加 `"ADMIN".equals(u.getRole())` 判断
  - [ ] JSP 菜单改为 `${sessionScope.user.role == 'ADMIN'}` 条件
  - [ ] LogoutServlet 改为 `session.invalidate()`
- [ ] 验证修复后6项测试全部通过：
  - [ ] 未登录访问 /admin/panel → 拦截
  - [ ] tom/USER 访问 /admin/panel → 403/拒绝
  - [ ] admin/ADMIN 访问 /admin/panel → 正常进入
  - [ ] tom/USER 首页 → 无管理后台入口
  - [ ] admin/ADMIN 首页 → 显示管理后台入口
  - [ ] 退出后再访问 → 被拦截
- [ ] 修复版作为录课备用，万一现场修改出错可快速切换

## 阶段二：录制课堂教学实录视频（40-50分钟）

- [ ] 确认两个机位：

  - 副机位：学生电脑操作+小组讨论近景
- [ ] 确认录制设备：视频MP4格式、720P以上、声音清晰
- [ ] 确认文件大小不超过1200MB
- [ ] 提前在教师电脑部署好问题版工程，确认可运行

- [ ] 安排学生分为A/B/C三组，每组明确任务

### 2.2 录制要点（对照逐分钟讲稿执行）
- [ ] 0-5min：教师演示问题系统，制造认知冲突

- [ ] 13-20min：学生分组使用AI审查代码（**必须拍到学生操作AI**）
- [ ] 20-26min：各组汇报AI发现+教师知识点对照
- [ ] 26-30min：教师系统提升（板书5大知识点）
- [ ] 30-35min：学生修改关键代码
- [ ] 35-39min：AI生成测试清单+学生逐项验证
- [ ] 39-42min：展示修复结果，三条路径验证
- [ ] 42-45min：课堂总结+课后拓展任务

### 2.3 录制后检查
- [ ] 检查视频是否完整、连续、无中断
